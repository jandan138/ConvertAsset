glTF metallic-roughness

glTF 2.0 是 Khronos 设计的「3D 运行时资产交换格式」（runtime delivery format）。它的材质系统以 pbrMetallicRoughness 为核心，专门采用 metallic-roughness workflow，并为纹理打包格式规定了严格的通道布局。ConvertAsset 的 export-glb 子命令把 USD 材质转换到这个格式。

From GAMES101 glTF 的材质模型和 UsdPreviewSurface 在数学上几乎一致——都是 GGX + Schlick + Smith，metalness workflow。区别主要在数据格式：颜色空间约定、纹理通道打包、JSON 结构。

pbrMetallicRoughness 结构

glTF 材质的 JSON 结构如下（节选）：

material.gltf（JSON 节选）
{
  "materials": [{
    "name": "GoldMetal",
    "pbrMetallicRoughness": {
      "baseColorFactor": [1.0, 0.71, 0.29, 1.0],
      "metallicFactor": 1.0,
      "roughnessFactor": 0.18,
      "baseColorTexture": { "index": 0 },
      "metallicRoughnessTexture": { "index": 1 }
    },
    "normalTexture": { "index": 2, "scale": 1.0 },
    "occlusionTexture": { "index": 3, "strength": 1.0 },
    "emissiveFactor": [0.0, 0.0, 0.0]
  }]
}

各参数语义：

  • baseColorFactor：albedo 的 RGBA 乘数，与 baseColorTexture 相乘。颜色通道 RGB 在 sRGB 色彩空间中；A 通道是 opacity。
  • metallicFactor：metallic 的标量乘数 \([0, 1]\)，与 metallicRoughnessTexture 相乘。
  • roughnessFactor：roughness 的标量乘数 \([0, 1]\)，与 metallicRoughnessTexture 相乘。

metallicRoughnessTexture 通道打包

glTF 把 metallic 和 roughness 打包进同一张纹理，以节省纹理单元：

  • Green 通道 → roughness 值（\([0,1]\)）
  • Blue 通道 → metallic 值（\([0,1]\)）
  • Red 通道保留，通常未使用（有些工具用于 occlusion，但 glTF spec 推荐用独立的 occlusionTexture）
Green = roughness，Blue = metallic 这一通道分配在 ConvertAsset 的 glb/writer.py 中有明确实现。写错通道（比如把 metallic 放 R 或 G）是很常见的 bug。记住口诀：G = roughness（粗糙），B = metallic（金属）。

sRGB vs 线性：必须正确

glTF 对纹理的颜色空间有严格规定：

  • baseColorTexture：sRGB——在采样时由 GPU 自动做 sRGB → linear 解码，然后参与光照计算。这与艺术家在 DCC 里「看到」的颜色一致。
  • metallicRoughnessTexture：linear——不做颜色空间转换，直接作为线性数据使用。如果错误地标记为 sRGB，粗糙度和金属度的数值会被 gamma 解码扭曲，导致材质外观完全错误。
  • normalTexture：linear（raw），同样不做 sRGB 解码。
  • occlusionTexture：linear。

其他纹理

除了 pbrMetallicRoughness 之外，glTF 材质还支持：

  • normalTexture：切线空间法线贴图（RGB，linear）。
  • occlusionTexture：AO 纹理（R 通道，linear）；有 strength 乘数。
  • emissiveTexture + emissiveFactor：自发光（sRGB 纹理 × linear 因子）。
  • 扩展（extensions）：如 KHR_materials_clearcoat、KHR_materials_transmission 等通过扩展机制增加额外属性。

ConvertAsset 中的 glTF 材质提取

ConvertAsset 的 glb/usd_material.py 模块从 UsdPreviewSurface 提取材质数据，glb/writer.py 将其写入 glTF JSON。关键映射：

gltf_material_json.json
{
  "pbrMetallicRoughness": {
    "baseColorFactor": [0.95, 0.78, 0.34, 1.0],
    "metallicFactor": 1.0,
    "roughnessFactor": 0.18
  }
}

对应 UsdPreviewSurface 参数：diffuseColor → baseColorFactor，metallic → metallicFactor，roughness → roughnessFactor。由于 USD 和 glTF 都在 renderer 内部做 \(\alpha = \text{roughness}^2\)，roughness 数值直接传递，无需转换。
