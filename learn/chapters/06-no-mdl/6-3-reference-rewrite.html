path / reference 重写

仅转换材质网络还不够——一个 USD 场景通常是多文件的：顶层 USD 通过 references / payloads / sublayers 引用子 USD，子 USD 再引用更深的文件。如果只把顶层材质换掉，子文件里的 MDL 仍然存在，渲染结果不变。references.py 与 processor.py 共同解决这个问题：递归处理整个文件树，并用 *_noMDL 后缀重写所有组合关系路径。

非打平策略（non-flatten composition-preserving）

最简单的做法是把整个场景"打平"（flatten）成一个单层 USDA，然后一次性替换所有材质。ConvertAsset 故意不这样做，原因：

  • 打平会失去所有 variant sets、reference / payload 结构——资产失去了可编辑性和 LOD / streaming 能力。
  • 打平会内联所有子文件的内容，输出文件可能几十倍膨胀。
  • 打平破坏了 USD 的组合哲学（"non-destructive opinion layering"）。

取而代之，Processor 为每个 USD 文件生成一个兄弟 *_noMDL.usd，内容是 root layer 的副本（仅 root layer，不内联子内容），然后把其中的路径引用改写为指向对应的 *_noMDL.usd，再执行材质转换。

路径收集：_collect_asset_paths

references.py 中的 _collect_asset_paths(stage) 扫描 Stage，收集六种类型的资产路径条目，每条返回六元组 (kind, holder, layer_dir, assetPath, prim_path, extra)：

  • "sublayer"——root layer 的 subLayerPaths。
  • "reference"——prim 上的 references metadata ListOp 中的每个 Sdf.Reference。
  • "payload"——prim 上的 payloads / payload ListOp 中的每个 Sdf.Payload。
  • "variant_ref" / "variant_payload"——进入每个 variant 编辑上下文后，variant 内的 references/payloads。
  • "clip_asset" / "clip_manifest"——valueClips 中的 clipAssetPaths 和 manifestAssetPath。
layer_dir 锚点规则 每条路径条目携带其所在 layer 的目录（layer_dir）。改写时，相对路径先以 layer_dir 为锚点绝对化，找到对应的 *_noMDL.usd 目标路径，再转回相对于各自 layer 的相对路径写回。这保持了工程的相对引用习惯，避免把所有路径写死为绝对路径。

路径改写：_rewrite_assets_in_stage

_rewrite_assets_in_stage(stage, mapping) 把 mapping（src_abs → dst_abs）应用到 Stage 上，逐类型改写：

references.py — 改写 sublayers（示意）
from pxr import Usd, Sdf
import os

def _rewrite_assets_in_stage(stage: Usd.Stage, mapping: dict):
    layer = stage.GetRootLayer()
    ldir  = os.path.dirname(layer.realPath or layer.identifier)

    # 1) sublayers
    new_subs = []
    for sl in layer.subLayerPaths:
        abs_sl = _resolve(ldir, sl)
        if abs_sl in mapping:
            new_subs.append(_relpath(ldir, mapping[abs_sl]))
        else:
            new_subs.append(sl)
    layer.subLayerPaths = new_subs

    # 2) prim references / payloads（通过 Sdf.ChangeBlock 批量更新）
    with Sdf.ChangeBlock():
        for spec in _iter_prim_specs(layer):
            _rewrite_refs(spec, mapping, ldir)
            _rewrite_payloads(spec, mapping, ldir)
            _rewrite_variants(spec, mapping, ldir)
            _rewrite_clips(spec, mapping, ldir)

去重与环检测

Processor 维护两个成员：

  • self.done: dict[str, str]——已处理的"源绝对路径 → *_noMDL 绝对路径"映射。同一文件第二次被引用时直接复用结果，不重复处理。
  • self.in_stack: set[str]——当前递归栈中的路径集合。若某文件在处理过程中又被自己间接引用（循环引用），则检测到并打印 [WARN] cyclic reference?，返回已知结果（或路径本身），防止无限递归。
小结 path / reference 重写的本质是：在不打平 USD 组合结构的前提下，把"原始文件树"映射为一棵结构同构的"_noMDL 文件树"，每个节点只更新自己的路径引用和材质网络，子节点由递归处理。这保证了 variant sets、references、payloads 等组合关系的完整性。
