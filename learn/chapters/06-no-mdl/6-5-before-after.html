PART II · 材质与外观 · Materials & Appearance

before / after 实测

理论讲完了，这一节看实际效果：no-MDL 转换到底保留了什么，丢失了什么，以及怎样用 ConvertAsset 的 CLI 跑一遍并验证结果。理解这张"得失清单"，是判断一个资产是否适合 no-MDL 工作流的重要参考。

转换中保留的内容

ConvertAsset 的非打平设计保证了以下内容在转换后完整保留：

  • 几何结构：所有 Mesh prim、points / faceVertexIndices / counts / normals / UV (st) 完全不变。
  • USD 组合关系：references、payloads、sublayers、variant sets、clips 的结构和顺序不变，仅路径后缀改为 _noMDL。
  • 场景层次：Xform 层次、material:binding 关系、light prims、camera prims 均不变。
  • BaseColor 贴图：如果 MDL Shader 有 inputs:diffuse_texture 连接，会被复制到 UsdPreviewSurface 的 inputs:diffuseColor UsdUVTexture 节点。这是视觉保真度最高的通道。
  • Roughness / Metallic 贴图或常量：OmniPBR 系 MDL 的标准参数会被正确提取和映射。
  • Normal 贴图：如果 MDL Shader 上有 normal map 输入，会连接到 PreviewSurface 的 inputs:normal。

转换中近似或丢失的内容

有损近似（lossy） no-MDL 转换是有损的。以下内容无法精确映射：
  • 复杂 BSDF 组合：多层 fresnel_layer + weighted_layer 构成的分层材质（如车漆、多层涂层）——UsdPreviewSurface 只有单层 metallic-roughness 模型，无法表达层间 Fresnel 权重变化。
  • 各向异性光泽（anisotropic glossy）：MDL 的 simple_glossy_bsdf 支持 roughness_u / roughness_v 各向异性，UsdPreviewSurface 只有单一 roughness 标量。
  • 次表面散射（SSS）：通过 VDF + BSDF 组合实现的皮肤/玉石效果完全丢失，降级为不透明漫反射。
  • 各向同性体积（volume）：material_volume 的吸收系数和散射系数无对应 UsdPreviewSurface 字段。
  • 程序化纹理：MDL 里由过程式函数生成的纹理（噪声、渐变、棋盘格等）——转换器不执行函数，只能用常量或不连接该通道。
  • 自发光细节：MDL EDF 中精细的方向性发光分布（spot_edf、measured_edf）——近似为 UsdPreviewSurface 的 emissiveColor 常量或贴图。
  • Opacity / cutout：MDL 的 geometry.cutout_opacity 目前仅做简单映射，不支持透明贴图的全通道混合。

CLI 操作实测

用 ConvertAsset 做 no-MDL 转换只需一条命令：

shell
# 基本用法：处理一个 USD，在同目录生成 *_noMDL.usd
./scripts/isaac_python.sh ./main.py no-mdl /path/to/scene.usd

# 输出示例（PRINT_DIAGNOSTICS_LEVEL=1）：
# [DONE] scene.usd -> scene_noMDL.usd | mats 12/15 noMDL=True (external-only-pass)

# 验证 PreviewSurface 结构
./scripts/isaac_python.sh ./main.py inspect /path/to/scene_noMDL.usd usdpreview /Looks/MyMaterial

转换完成后，Processor 会输出一行日志：

日志含义解读
[DONE] scene.usd -> scene_noMDL.usd | mats 12/15 noMDL=True (external-only-pass)
#                                              ↑  ↑                ↑
#                                      已转换  总数   reason 说明符

# reason 可能取值：
# "strict-pass"          — 所有 MDL 全部清除
# "external-only-pass"   — root 层清除，外部引用残留 MDL（ALLOW_EXTERNAL_ONLY_PASS=True）
# "strict-fail"          — 仍有 MDL 残留（需人工排查）

Summary 与 Audit 文件

当 WRITE_SUMMARY_TXT = True（默认开启）时，同目录还会生成 scene_noMDL_summary.txt，记录转换统计、缺失子文件列表、递归映射概览等信息，便于排查问题。WRITE_AUDIT_JSON = True 时额外生成结构化 JSON，用于程序化后处理和趋势分析。

关键配置开关
  • SUFFIX = "_noMDL"——输出文件后缀，可按需修改。
  • OVERRIDE_EXTERNAL_MDL_PREVIEW = True——在当前 layer 对外部材质创建 override + PreviewSurface，显著改善顶层查看视觉（但增加当前 layer 体积）。
  • ALLOW_EXTERNAL_ONLY_PASS = True——root 层已清除时即判成功，避免因无法处理的外部引用一直 fail。
  • DIAGNOSTIC_LEVEL = 2（即 PRINT_DIAGNOSTICS_LEVEL）——控制终端日志详细程度：0=极简，1=关键信息，2=全部 DIAG 输出。
第 6 章总结 no-MDL 转换是 ConvertAsset 的核心工作流：把 renderer-bound 的 MDL 材质近似转换为 portability 友好的 UsdPreviewSurface，同时保留 USD 的完整组合结构。算法分四步（定位→提取→构建→清理），通过递归处理整个文件树并重写 *_noMDL 路径完成。论文从 perturbation 视角量化了这个转换的影响边界：proxy 指标高（PSNR 35.52 dB，SSIM 0.990），但 proxy 忠诚度不等同于下游 grounding 任务的无影响——需要四层证据门分别验证。
