VLN 任务是什么

VLN（Vision-Language Navigation，视觉语言导航）是具身 AI 领域的经典任务：给定自然语言指令（如"走到厨房，找到红色烤箱旁边的椅子"），让 agent 在 3D 室内环境里自主导航到目标位置。它把计算机视觉、自然语言理解和序列决策三个领域紧密耦合在一起。

VLN 任务定义

一个标准的 VLN episode 由以下要素组成：

  • 环境：3D 室内场景，通常表示为可导航图（navigation graph）或连续空间（continuous environment）。
  • 指令：自然语言描述的导航路径，通常由人工标注，包含地标、方向、动作序列。
  • 观测：agent 在当前位置的 360° 全景图（panorama）或 egocentric 视图，即 agent 的"眼睛"所看到的 RGB 图像。
  • 动作空间：离散（选择下一个 viewpoint 节点）或连续（速度向量）。
  • 成功判据：agent 停在距目标 $d$ 米以内（通常 $d = 3$ 米），用 Success Rate (SR) 和 SPL（Success weighted by Path Length）衡量。

典型 benchmark

VLN 领域最常引用的 benchmark 按发布年代列举如下：

  • R2R（Room-to-Room，2018）：Matterport3D 扫描场景，离散导航图，成为领域标准基准。
  • REVERIE（2020）：在 R2R 基础上增加物体定位，要求 agent 同时导航并定位目标物体。
  • VLNVerse（2024）：把多个 VLN 任务（R2R / CVDN / ScanQA 等）统一到一个 framework，并支持 Isaac Sim 仿真场景。
  • InternNav / InteriorAgent：基于 NVIDIA Isaac Sim 的室内导航 benchmark，使用 KuJiaLe（酷家乐）场景，支持 MDL / UsdPreviewSurface 材质切换，是 ConvertAsset 论文使用的具身仿真平台。
ConvertAsset 与 InternNav 的关系 ConvertAsset 的论文在 InternNav 的官方 KuJiaLe 场景上运行了 99 episode（val_unseen 集）的 sanity run，对比 original MDL 和转换后 noMDL 的 SR / SPL / NE / TL。结果显示，noMDL 与 original 的差异在统计误差范围内（详见 14-4 节 evidence-gate 分析）。

仿真与真实世界的迁移

VLN 研究面临的核心挑战之一是 sim-to-real gap：在仿真环境（如 Isaac Sim 渲染的场景）训练的策略，在真实机器人上部署时性能会下降。影响因素包括：

  • 视觉保真度：仿真场景的材质、光照与现实的差距（domain gap）。材质转换对视觉保真度的影响是 ConvertAsset 定量研究的核心。
  • 物理保真度：摩擦、碰撞响应的差异影响运动控制策略。
  • 语言 grounding：室内场景里的物体外观（颜色、材质）是语言指令的关键线索。如果材质转换改变了物体的视觉特征，VLM 对指令的解析可能出错——这正是 14-5 节讨论的 material-effect risk。
SPL 公式 SPL（Success weighted by Path Length）定义为： $$ \text{SPL} = \frac{1}{N} \sum_{i=1}^{N} S_i \cdot \frac{l_i}{\max(p_i, l_i)} $$ 其中 $S_i \in \{0,1\}$ 是是否成功，$l_i$ 是最短路径长度，$p_i$ 是实际走过的路径长度。SPL 惩罚绕路，奖励高效导航。
