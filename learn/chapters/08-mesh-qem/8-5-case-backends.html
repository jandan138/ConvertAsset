case：simplify.py 三个 backend 与 UV 保持

ConvertAsset 提供三种网格简化 backend，从纯 Python 到 C++ pybind11，覆盖"开箱即用"到"生产性能"的需求谱系。最关键的工程挑战是：在几何拓扑被改写的同时，如何保持 face-varying UV 的正确对应——ConvertAsset 用"面级 UV 三元组随面存活"的方案解决了这个问题。

三个 backend 概览

Backend CLI 参数 核心文件 UV 保持 适用场景
py --backend py mesh/simplify.py ✓（qem_simplify_ex） 无编译依赖，小中型网格
cpp --backend cpp mesh/backend_cpp.py + 外部可执行 ✗（OBJ 不含 UV） 大网格、无需 UV 精度
cpp-uv --backend cpp-uv mesh/backend_cpp.py + pybind11 ✓（内存传递 UV triplets） 生产性能 + UV 精度

py backend：qem_simplify_ex

纯 Python backend 的核心是 qem_simplify_ex 函数，相比基础版 qem_simplify 多了一个 face_uvs 参数——每个三角形对应一个 UV 三元组（6 个浮点：\(u_0,v_0,u_1,v_1,u_2,v_2\)）：

mesh/simplify.py（qem_simplify_ex 核心）
def qem_simplify_ex(verts, faces, target_faces,
                    face_uvs=None, ...):
    """QEM 简化 + face-varying UV 同步追踪"""
    F = [list(f) for f in faces]          # 可变三角面副本
    f_alive = [True] * len(F)             # 面存活标记
    # ...（初始化 quadric、堆等）...

    # 主坍塌循环
    while faces_current > faces_target and heap:
        cost, _, u, v, pos = heapq.heappop(heap)
        # ...（执行坍塌，更新 F[fi]）...
        for fi, f in enumerate(F):
            if not f_alive[fi]: continue
            # 把 v 替换为 u，若三顶点重复则面退化
            a,b,c = f
            if a==v: a=u
            if b==v: b=u
            if c==v: c=u
            if len({a,b,c}) < 3:
                f_alive[fi] = False   # 面退化
                faces_current -= 1
            else:
                F[fi] = [a,b,c]

    # 压缩阶段：只保留存活的面和对应 UV
    new_face_uvs = []
    for fi, alive in enumerate(f_alive):
        if not alive: continue
        a,b,c = F[fi]
        if a in index_map and b in index_map and c in index_map:
            new_faces.append((index_map[a],index_map[b],index_map[c]))
            if face_uvs is not None:
                new_face_uvs.append(face_uvs[fi])  # UV 与面一一对应

关键设计：UV 三元组与面索引 fi 严格绑定——面退化时 UV 自动丢弃，面存活时 UV 随面保留。坍塌操作不修改 UV 值，只影响哪些面存活。这保证了 UV 语义的正确性（每个三角的角点 UV 不因顶点位置移动而改变）。

cpp backend：临时 OBJ 文件路径

simplify_mesh_with_cpp 把 USD mesh 提取为 OBJ，调用外部 meshqem 可执行文件，再读回 OBJ 写入 USD：

mesh/backend_cpp.py（节选）
def simplify_mesh_with_cpp(prim, exe_path, ratio=None, ...):
    points, faces = extract_tri_mesh_from_prim(prim)
    with tempfile.TemporaryDirectory() as td:

这条路径不传递 UV：OBJ 格式理论上支持 UV，但 meshqem 的 OBJ 接口只处理几何。适合只需减面、不关心 UV 精度的场景（如碰撞网格）。

cpp-uv backend：pybind11 内存传递

simplify_mesh_with_cpp_uv 通过 pybind11 模块 meshqem_py 直接在 Python 与 C++ 之间传递顶点、面和 UV 三元组，避免磁盘 I/O：

mesh/backend_cpp.py（cpp-uv 路径）
def simplify_mesh_with_cpp_uv(prim, ratio=None, ...):
    if meshqem_py is None:
        raise RuntimeError("meshqem_py binding not available")
    points, faces = extract_tri_mesh_from_prim(prim)
    face_uv_triplets = extract_facevarying_uv_triplets(prim)

    # 直接调用 C++ 函数，内存传递
    new_verts, new_faces, new_face_uvs = meshqem_py.simplify_with_uv(
        points, faces, face_uv_triplets, ratio, ...)

    if apply:
        mesh.GetPointsAttr().Set(new_verts)
        # 写回拓扑...
        if new_face_uvs is not None:
            _write_facevarying_uv(mesh, new_face_uvs, name="st")

架构图：三 backend 共享的数据流

工程小结 py backend 是默认路径，开箱即用，UV 正确，适合中小型网格或调试。cpp backend 速度最快但丢失 UV，用于碰撞体生成。cpp-uv 是生产推荐——C++ 速度 + UV 完整性，需要先编译 pybind11 扩展。三条路径共享同一套 USD 读写接口，仅"计算核"不同。
