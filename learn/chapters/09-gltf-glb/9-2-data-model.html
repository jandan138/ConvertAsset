scene / node / mesh / accessor / bufferView / buffer

glTF 的数据模型由两层构成：场景树（scene → node → mesh）描述"什么东西在哪里"；二进制存储栈（accessor → bufferView → buffer）描述"数据怎么存"。理解这六层，就能理解 ConvertAsset 的 glb/writer.py 是怎么把 USD 数据打包进 GLB 的。

场景树：scene / node / mesh

scene 是 glTF 文件的顶层入口，包含一组根 node 的索引列表。

node 是场景图的基本单元，携带变换（translation / rotation / scale 或 4×4 matrix），并可引用一个 mesh、一个摄像机、或一个蒙皮（skin）。Node 可以有子节点，组成树形层级。

mesh 是几何体的容器，由一组 primitives 组成。每个 primitive 对应一个绘制调用：

mesh_primitive.json（示意）
{
  "primitives": [{
    "attributes": {
      "POSITION":   0,   // accessor 索引
      "NORMAL":     1,
      "TEXCOORD_0": 2
    },
    "indices":   3,      // accessor 索引
    "material":  0,
    "mode":      4       // 4 = TRIANGLES
  }]
}

二进制存储栈：accessor / bufferView / buffer

glTF 把所有数值数据（顶点坐标、UV、法线、索引……）存进二进制块，通过三层间接寻址访问：

层 描述什么 关键字段
buffer 原始二进制 blob（可以是 Base64 内嵌或外链 .bin 文件） byteLength, uri
bufferView buffer 里的一个连续字节范围（"切片"），可设 target 提示 GPU 绑定目标 buffer, byteOffset, byteLength, target
accessor 如何解读 bufferView 里的字节：类型（VEC3/SCALAR…）、分量类型（float/ushort…）、元素数量 bufferView, byteOffset, componentType, type, count, min, max

比如，1000 个顶点的 POSITION accessor 对应 12000 字节（1000 × 3 × 4 字节 / float），在 bufferView 里偏移到某个字节范围，bufferView 再指向某个 buffer blob。

完整小例子

triangle.gltf（最小 glTF）
{
  "scene": 0,
  "scenes": [{ "nodes": [0] }],
  "nodes":  [{ "mesh": 0 }],
  "meshes": [{
    "primitives": [{
      "attributes": { "POSITION": 0 },
      "indices": 1
    }]
  }],
  "accessors": [
    {
      "bufferView": 0, "componentType": 5126,
      "count": 3, "type": "VEC3",
      "min": [-0.5,-0.5,0], "max": [0.5,0.5,0]
    },
    {
      "bufferView": 1, "componentType": 5123,
      "count": 3, "type": "SCALAR"
    }
  ],
  "bufferViews": [
    { "buffer": 0, "byteOffset": 0,  "byteLength": 36, "target": 34962 },
    { "buffer": 0, "byteOffset": 36, "byteLength": 6,  "target": 34963 }
  ],
  "buffers": [{ "byteLength": 44, "uri": "triangle.bin" }]
}

这里 componentType: 5126 = GL_FLOAT，5123 = GL_UNSIGNED_SHORT；target: 34962 = ARRAY_BUFFER（顶点缓冲），34963 = ELEMENT_ARRAY_BUFFER（索引缓冲）。这些常数直接来自 OpenGL ES 2.0 规范。

小结 glTF 的六层结构把"逻辑描述"（scene/node/mesh）和"物理存储"（accessor/bufferView/buffer）分离。引擎加载时只需把 buffer 直接 DMA 到 GPU，非常高效。9-3 讲 GLB 如何把 JSON 和 buffer 打包进一个二进制文件。
