GLB 二进制容器

GLB（GL Binary）是 glTF 的"打包版"：把原本分散的 JSON 文件、二进制缓冲区（.bin）和纹理图片合并进一个单一的二进制文件。它的格式极其简洁——12 字节头 + 若干 chunk——让任何语言都能在几十行代码里读写一个合法的 GLB。

文件 Header（12 字节）

GLB 文件以固定的 12 字节头开始，全部 little-endian：

偏移 字段 大小 值
0 magic uint32 0x46546C67（ASCII "glTF"，little-endian）
4 version uint32 2（glTF 2.0）
8 length uint32 整个 GLB 文件的字节长度

magic 的 ASCII 字节序是 67 6C 54 46（小端存储），用 xxd output.glb | head -1 可以直接看到开头 "glTF"。

Chunk 结构

Header 之后是一系列 chunk，每个 chunk 有 8 字节元数据 + 内容：

偏移（chunk 内） 字段 大小 说明
0 chunkLength uint32 chunk 内容字节数（不含 8 字节元数据）
4 chunkType uint32 0x4E4F534A = JSON；0x004E4942 = BIN\0
8 chunk data chunkLength 字节 内容；必须 4 字节对齐（填充）

规范要求：第一个 chunk 必须是 JSON chunk（type 0x4E4F534A），用空格（0x20）填充到 4 字节对齐；第二个 chunk（若有）是二进制 BIN chunk（type 0x004E4942），用零（0x00）填充到 4 字节对齐。ConvertAsset 只使用这两个 chunk。

ConvertAsset writer.py 的写出逻辑

ConvertAsset 的 glb/writer.py 用纯 Python struct 模块写出 GLB，核心步骤是：

  1. 把所有网格数据（positions, normals, uvs, indices）打包成二进制 BIN blob。
  2. 构建 JSON 字典（scenes / nodes / meshes / accessors / bufferViews / buffer / materials）。
  3. 序列化 JSON → bytes，4 字节对齐（空格填充）。
  4. 计算 total length = 12 + 8 + len(json_bytes) + 8 + len(bin_bytes)。
  5. 依次写入：header、JSON chunk header + 内容、BIN chunk header + 内容。

互动：GLB 二进制 chunk 解剖

互动 · Interactive
GLB 二进制 chunk 解剖
上传或选择一个 .glb 文件，互动面板会解析并高亮展示 header、JSON chunk 与 BIN chunk 的边界，列出每个 accessor 的 byteOffset 和字节范围。
小结 GLB = 12B Header + JSON Chunk + BIN Chunk。Magic 0x46546C67（"glTF"），version = 2，chunk 必须 4 字节对齐。ConvertAsset 的 writer.py 只用这个最简结构，生成最大兼容性的 GLB，不依赖任何第三方 glTF 库。
