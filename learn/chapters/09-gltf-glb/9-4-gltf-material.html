PBR material 在 glTF 里的表达

glTF 2.0 的核心材质模型是 pbrMetallicRoughness——基于 Cook-Torrance 微平面理论的 metallic-roughness workflow。它把 PBR 所需的所有参数（base color、metallic、roughness、法线贴图、自发光……）打包进一个紧凑的 JSON 对象，与你在第 3–4 章学到的 UsdPreviewSurface 参数体系高度对齐。

From GAMES101 Cook-Torrance 的 D / F / G 三项（见 3-4）需要 roughness 和 metallic 两个关键输入。glTF 的 pbrMetallicRoughness 正是把这两个参数（以及 base color）规范化成一个跨平台标准，任何支持 glTF 的引擎都用同样的公式解释这些值。

pbrMetallicRoughness 字段详解

material.json（glTF 材质示例）
{
  "name": "GoldMetal",
  "pbrMetallicRoughness": {
    "baseColorFactor": [0.95, 0.78, 0.34, 1.0],  // RGBA，线性空间
    "baseColorTexture": { "index": 0 },           // 可选 base color 贴图
    "metallicFactor": 1.0,                        // 0=非金属, 1=金属
    "roughnessFactor": 0.18,                      // 0=镜面, 1=完全漫反射
    "metallicRoughnessTexture": { "index": 1 }    // G=roughness, B=metallic（ORM 布局）
  },
  "normalTexture": {
    "index": 2,
    "scale": 1.0
  },
  "emissiveFactor": [0.0, 0.0, 0.0],
  "doubleSided": false,
  "alphaMode": "OPAQUE"
}

关键参数语义

参数 范围 语义 对应 UsdPreviewSurface
baseColorFactor [0,1]⁴ diffuse/albedo 颜色，线性空间 RGBA diffuseColor
metallicFactor [0,1] 金属度（影响 F0 和 diffuse 贡献） metallic
roughnessFactor [0,1] 粗糙度（GGX α = roughness²） roughness
metallicRoughnessTexture — ORM 贴图：R=occlusion, G=roughness, B=metallic 分别对应多个 texture 输入
normalTexture — 切线空间法线贴图（RGB = 切线空间 XYZ） normal
emissiveFactor [0,1]³ 自发光颜色，线性 RGB emissiveColor

ORM 贴图打包约定

glTF 规范把 occlusion（遮蔽）、roughness、metallic 三张灰度图打包进同一张 RGB 纹理的三个通道，节省采样器数量。约定：

  • R 通道：ambient occlusion（0=完全遮蔽，1=无遮蔽）
  • G 通道：roughness
  • B 通道：metallic

这与 Unreal Engine 的 ORM（Occlusion-Roughness-Metallic）约定相同，但与 Unity 的 RGBA 打包不同——导入引擎时需确认通道映射。

透明度模式

glTF 支持三种 alphaMode：

  • OPAQUE（默认）：完全不透明，忽略 alpha。
  • MASK：alpha cutout，低于 alphaCutoff（默认 0.5）的片元被丢弃。
  • BLEND：alpha 混合，需要正确的透明排序。

ConvertAsset 当前只输出 OPAQUE 模式，不处理半透明材质。

小结 glTF 的 PBR 材质本质是对 Cook-Torrance metallic-roughness workflow 的标准化 JSON 表达。UsdPreviewSurface 与 glTF PBR 的参数一一对应，这是 ConvertAsset 能做无损材质转换的数学基础——只要 no-MDL 阶段把 MDL 正确映射到 UsdPreviewSurface，glTF 导出就能保留所有 PBR 信息。
