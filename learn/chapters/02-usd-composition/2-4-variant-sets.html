variant sets

VariantSet 让资产创作者把「同一资产的不同版本」打包进一个文件，并暴露一个可供消费者切换的「变体选择器」。下游用户可以在 stronger layer 里声明 variants = { string material = "gold" }，即可非破坏地切换到金色版本，不需要任何 if/else 代码，也不需要复制资产文件。

From GAMES101 在 GAMES101 里，你如果想让同一张桌子有木纹和金属两个版本，通常是在代码里用 if 分支加载不同的材质文件。USD 的 VariantSet 把这种「运行时分支」变成「数据声明」：变体本身是数据，切换是数据覆盖，而不是代码逻辑。

创作 VariantSet

在 USDA 文本格式里，VariantSet 的结构很直观：

chair_variants.usda
#usda 1.0
def Mesh "Chair" (
    # 声明拥有一个名叫 "material" 的 VariantSet
    prepend variantSets = "material"
    # 默认选中 "wood" 变体
    variants = {
        string material = "wood"
    }
) {
    # base 数据（所有变体共享）
    int[] faceVertexCounts = [4, 4, 4, 4, 4, 4]
    int[] faceVertexIndices = [0, 1, 2, 3, 4, 5, 6, 7, ...]
    point3f[] points = [(-0.5, 0, -0.5), ...]

    variantSet "material" = {
        "wood" {
            # 木纹变体：覆盖材质绑定
            rel material:binding = 
        }
        "metal" {
            # 金属变体：不同材质
            rel material:binding = 
        }
        "fabric" {
            # 布料变体
            rel material:binding = 
        }
    }
}

在 stronger layer 切换变体

消费者在自己的 layer（比 asset 更强）里切换变体，不需要改动 asset 文件：

room.usda（切换变体）
#usda 1.0
def Xform "Room" {
    def Xform "Chair" (
        references = @./chair_variants.usd@
        # L 层：直接 override variant selection
        variants = {
            string material = "metal"  # 切换到金属版本
        }
    ) { }
}

注意：variants 是一个 Local opinion（L 层），比 VariantSet 本身（V 层）更强——所以即使 chair.usd 里默认选了 "wood"，room.usda 里的 Local override 会让 "metal" 生效。

多个 VariantSet 沿独立轴变化

一个 Prim 可以有多个 VariantSet，分别沿不同的轴变化：

chair_multi_variant.usda
#usda 1.0
def Mesh "Chair" (
    prepend variantSets = ["material", "lod"]
    variants = {
        string material = "wood"
        string lod      = "high"
    }
) {
    variantSet "material" = {
        "wood"   { rel material:binding =  }
        "metal"  { rel material:binding =  }
    }
    variantSet "lod" = {
        "high"   {
            # 高细节 mesh
            int[] faceVertexCounts = [4, 4, 4, 4, 4, 4, 4, 4]
        }
        "low"    {
            # 低细节 mesh（简化版）
            int[] faceVertexCounts = [4, 4, 4, 4]
        }
    }
}

两个 VariantSet（material × lod）可以独立切换，组合出 2×2 = 4 种配置，全部来自同一个 USD 文件。

用 pxr API 操作 VariantSet

variant_api.py
from pxr import Usd

stage = Usd.Stage.Open("chair_variants.usd")
prim  = stage.GetPrimAtPath("/Chair")

# 列出所有 VariantSet
vsets = prim.GetVariantSets()
print("variant sets:", vsets.GetNames())  # ['material']

# 获取某个 VariantSet
mat_vset = vsets.GetVariantSet("material")
print("variants:", mat_vset.GetVariantNames())   # ['wood', 'metal', 'fabric']
print("selection:", mat_vset.GetVariantSelection())  # 'wood'

# 切换变体（在 stage 的 edit target 上写 opinion）
mat_vset.SetVariantSelection("metal")

# 在特定 variant 的上下文里 author 内容（创作时）
with mat_vset.GetVariantEditContext():

    metal_mat = stage.GetPrimAtPath("/Looks/MetalMat")
