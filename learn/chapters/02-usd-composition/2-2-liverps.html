LIVERPS strength ordering

当多个 composition arc 都对同一个属性发表了 opinion，USD 必须决定哪个赢。这个裁决规则叫 LIVERPS（也写作 LIVRPS）：Local > Inherits > VariantSets > rEferences > Payloads > Sublayers。从左到右，强度依次递减。越靠近「你自己在当前文件里写的」就越强；越靠近「从外部引入的」就越弱。

From GAMES101 在 GAMES101 里你没有遇到「多个数据源的同一属性冲突」的问题——场景是单文件的，每个属性只有一个值。LIVERPS 是 USD 引入多文件、多覆盖层之后才需要的裁决机制。可以类比 CSS 的选择器优先级：!important > inline style > id > class > tag。LIVERPS 是 USD 的「CSS 优先级」。

LIVERPS 六层详解

L — Local（最强）
当前 Prim 在当前 Layer 里「亲自写下」的 opinion。这代表「你现在手里拿的这支笔写的字」，任何从外部引入的内容都无法覆盖它。
I — Inherits
通过 inherits arc 引入的 base Prim 的 opinion。比 Local 弱，但比任何 referenced 内容都强——这意味着你可以在一个 class Prim 上做改动，所有 inherit 它的 Prim 都会看到这个改动，哪怕那些 Prim 本身在 stronger layer 里有 override。
V — VariantSets
当前激活的 variant 里的 opinion。VariantSet 的设计意图是让变体「比 reference 内容更强」——你可以通过切换 variant 来覆盖 referenced asset 的某些属性，而不需要写 Local override。
E — rEferences（references）
通过 references arc 引入的外部 Prim 的 opinion。「把 chair.usd 放进 room.usd」这个 reference 里的所有数据，强度在此。
P — Payloads
与 references 相同的语义，但强度低于 references。实践中，同一个 Prim 很少同时有 reference 和 payload——通常资产选其一。
S — Sublayers（最弱）
通过 subLayers 引入的 Layer 里的 opinion。sublayer 是「协作覆盖」的机制，越靠前的 sublayer 越强；整个 subLayers 列表中，任何位置都比 references 弱。

一个具体例子

以下场景：room.usda 通过 reference 引入 chair.usd，同时有一个 override.usda sublayer。chair 内部有颜色变体。谁的 opinion 最终赢？

room.usda
#usda 1.0
(
    subLayers = [@./override.usda@]   # S 层
)

def Xform "Room" {
    def Xform "Chair" (
        references = @./chair.usd@    # E 层（chair 内容来自此）
        variants = { string color = "red" }   # V 层选择器
    ) {
        # L 层：Local override，最强
        # 如果这里写了 primvars:displayColor，它会覆盖一切
        # （此处省略，让 V 层的 variant 生效）
    }
}
chair.usd
#usda 1.0
def Mesh "Chair" (
    prepend variantSets = "color"
) {
    # E 层 base 颜色（白色）
    color3f[] primvars:displayColor = [(1, 1, 1)]

    variantSet "color" = {
        "red" {
            # V 层：variant 覆盖（比 E base 强，但比 L 弱）
            color3f[] primvars:displayColor = [(0.9, 0.1, 0.1)]
        }
        "blue" {
            color3f[] primvars:displayColor = [(0.1, 0.1, 0.9)]
        }
    }
}
override.usda
#usda 1.0
# S 层（sublayer）——比 reference 弱，比 variant 弱
over "Room" {
    over "Chair" {
        # 这个颜色 opinion 会被 V 层（variant）覆盖，因为 V > S
        color3f[] primvars:displayColor = [(0, 0, 1)]  # 蓝色
    }
}
# 最终结果：Chair 显示 red（V 层 variant 胜出），
#           不是蓝色（S 层 sublayer 输给了 V）
LIVERPS 的直觉 越「本地」的 opinion 越强。Local 是你自己写的；Inherits 来自 class；Variants 是当前文件选好的变体；References 是你引入的外部资产；Payloads 是懒加载版的 reference；Sublayers 是背景协作层。越靠右的来源越「背景化」，越靠左越「前景化」。

工程含义

LIVERPS 在工程实践中有几个直接含义：

  • Local override 是万能的：如果你在当前文件里 Local 写了一个属性值，没有任何 composition 机制能覆盖它（在这个 Layer 的视角里）。这是 ConvertAsset 转换材质的关键：转换后在 noMDL stage 里写入的 UsdPreviewSurface 属性是 Local opinion，会覆盖原来从 reference 带入的 MDL 属性。
  • VariantSet 可以覆盖 reference 内容：如果 chair.usd 里的 variant 设置了某个属性，它比 reference 的 base 内容更强——这就是 VariantSet 的设计价值：让资产自己管理自己的变体。
  • Sublayer 是最安全的「非侵入覆盖」：灯光师把调整写在 sublayer 里，比任何 reference 都弱，因此不会意外覆盖 asset 内部的变体选择。
