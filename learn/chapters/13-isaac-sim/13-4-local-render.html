单资产本地出图（render / camera）

ConvertAsset 的 render-single 子命令在本地 Isaac Sim 环境中对单个 USD 资产进行多视角出图，无需任何外部渲染服务。核心挑战是：如何在不知道资产朝向和尺寸的情况下，自动计算出"好看"的相机位置？答案是 bbox 驱动的 orbit 相机取景。

出图流程总览

整个流程分为六步，对应 convert_asset/render/single.py 的 run_render_single 函数：

bbox 计算与双策略验证

自动相机取景的核心是计算资产的世界空间包围盒（bounding box）。但 USD 资产的 extent 属性并不总是可靠——有时 extent 没有被更新，或者因为坐标变换而与实际网格偏离。

_compute_bbox 函数采用双策略验证：

  1. authored bbox（UsdGeom.BBoxCache）：从 Prim 的 extent 属性读取，快速但可能不准。
  2. mesh point bbox（_mesh_point_bbox）：遍历所有可见的 UsdGeom.Mesh，读取 points，用当前世界变换矩阵转换到世界空间后求 AABB，慢但精确。

两个 bbox 通过 _choose_bbox 裁定：如果 authored bbox 的对角线比 mesh bbox 的 5 倍还大（extent_fallback_ratio=5.0），说明 extent 属性已经过时，使用 mesh bbox；如果中心偏移超过 mesh 对角线的 1 倍（center_offset_threshold=1.0），同样切换到 mesh bbox。

render/single.py — _choose_bbox（节选）
authored_diagonal = np.linalg.norm(authored_bbox[1] - authored_bbox[0])
mesh_diagonal = np.linalg.norm(mesh_bbox[1] - mesh_bbox[0])

# authored bbox 比 mesh bbox 大 5 倍以上 → 说明 extent 过时
if authored_diagonal / mesh_diagonal >= extent_fallback_ratio:
    return mesh_bbox, "mesh_extent_ratio"

# 中心偏移超过 mesh 对角线 → authored bbox 位置错误
center_offset_ratio = np.linalg.norm(authored_center - mesh_center) / mesh_diagonal
if center_offset_ratio >= center_offset_threshold:
    return mesh_bbox, "mesh_center_offset"

return authored_bbox, "authored"

orbit 相机取景算法

确定了 bbox 中心和半径后，_set_camera_look_at 将相机放置在以中心为原点、半径为 distance 的球面上，均匀分布 sample_number 个方位角（azimuth），固定仰角（elevation，默认 35°）：

$$ \text{azimuth}_i = \text{azimuth\_offset} + i \times \frac{360°}{N}, \quad i = 0, 1, \ldots, N-1 $$

相机的空间位置由球坐标转笛卡尔坐标给出：

$$ \begin{pmatrix} x \\ y \\ z \end{pmatrix} = \text{target} + d \begin{pmatrix} \cos\theta_\text{elev} \cos\theta_\text{azim} \\ \cos\theta_\text{elev} \sin\theta_\text{azim} \\ \sin\theta_\text{elev} \end{pmatrix} $$

朝向（orientation）通过 scipy 的 Rodrigues 旋转给出，最终设置到 omni.isaac.sensor.Camera 的 world_pose。

更完整的 orbit 相机动画功能（含多垂直角度步、继承现有相机内参）在 camera/orbit.py 的 OrbitParams / create_orbit_camera_animation_and_export 实现；单帧取景拟合则在 camera/fit.py 的 FitParams / fit_camera_and_export 实现，支持继承场景现有相机的视角并用 Rodrigues 旋转做俯仰微调。

调用示例

CLI 用法
# 最简调用：对 chair.usd 从 4 个均匀方位角出图，512×512，PathTracing 渲染
./scripts/isaac_python.sh ./main.py render-single \
    /data/assets/chair.usd \
    --out /data/renders \
    --views 4 \
    --width 512 --height 512 \
    --elevation 35 \
    --renderer PathTracing

# 输出：/data/renders/chair/chair_0.png ... chair_3.png
# naming-style=view 时输出：front.png left.png back.png right.png
HDRI 环境光 _find_builtin_hdri 会自动搜索 Isaac Sim 安装目录下的 photo_studio_01_4k.hdr（omni.kit.widget.material_preview 扩展提供），如果找到，自动配置 DomeLight 并开启透明背景合成（/rtx/post/backgroundZeroAlpha）；找不到时退回灰色 DomeLight。
