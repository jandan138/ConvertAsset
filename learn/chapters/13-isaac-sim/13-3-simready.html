SimReady assets 与为什么要转换 / 简化

SimReady 是 NVIDIA 为仿真生产资产制定的一套规范：几何、物理、材质全部用 USD 表达，并附带正确的 pivot、单位、碰撞代理和 LOD 层级。然而现实中的资产往往并不符合所有要求——材质用了 MDL，网格面数过多，路径是绝对路径。ConvertAsset 正是为了弥合这个差距而存在的。

什么是 SimReady

NVIDIA SimReady 标准要求资产满足以下条件：

  • 几何：有碰撞代理网格（通常是简化的凸包或 convex decomposition），坐标轴原点在底面中心，单位为米，面法线向外。
  • 物理：附加 PhysicsRigidBodyAPI / PhysicsCollisionAPI，标注质量和摩擦系数。
  • 材质：使用 MDL 或 UsdPreviewSurface，纹理路径可解析。
  • LOD：提供多个细节等级，渲染网格与碰撞网格分离。
  • USD 结构：使用 references / variants 做变体管理，不要把所有内容都 flatten 到单一 Layer。

为什么要做 MDL → UsdPreviewSurface 转换

NVIDIA 官方资产库（如 Assets.nvidia.com）大量使用 MDL 材质，这些材质在 Omniverse RTX 渲染器下效果绝佳。但问题是：

  1. 可移植性（portability）：将资产导出为 GLB 时，glTF 格式不认识 MDL；Web 端、移动端、Blender、Unity 都无法正确显示材质。
  2. 合成数据稳定性：当用 Python 批量处理场景时，MDL 解析器如果遇到路径缺失或版本不匹配，会静默失败或输出错误颜色。UsdPreviewSurface 作为 OpenUSD 官方规范，具有更好的跨工具保证。
  3. 下游任务兼容：具身 AI 研究中，VLN / VLM 管线通常通过 Web 渲染器或 Three.js 展示场景，这些工具依赖 glTF metallic-roughness（即 UsdPreviewSurface 的映射目标）。
composition-preserving 的关键决策 ConvertAsset 的 no-mdl 转换是非破坏性的：它为每个 USD 文件生成一个 *_noMDL.usd 兄弟文件，保留原有的 references / variants / sublayers，只把材质网络替换为 UsdPreviewSurface。这样原始资产完整保留，可以随时对比。

为什么要做网格简化

建模师为了视觉精确度往往制作高面数网格，一件家具可能有 50 万到 200 万个三角形。在仿真场景中这带来两个问题：

  • 碰撞检测代价：PhysX 使用 convex decomposition 做碰撞，碰撞代理的面数与仿真步时间成正比。高精度碰撞网格会让 PhysX 步长拉长，实时性下降。
  • 渲染 batching：在合成数据批量生成时（Replicator / thumbnails 命令），场景里可能同时存在数十件家具。如果每件都是百万面，GPU 显存会迅速耗尽。

ConvertAsset 使用 QEM（Quadric Error Metrics，第 8 章）算法，在控制视觉误差的前提下把面数压缩到目标比例（如 --ratio 0.5 缩减到原来的 50%）。三个 backend——纯 Python（py）、C++ 可执行（cpp）、pybind11 C++（cpp-uv）——适用于不同规模和 UV 保持需求。

ConvertAsset 的完整流水线 一个典型的 SimReady 准备流程：
1. no-mdl：MDL → UsdPreviewSurface，生成 *_noMDL.usd
2. mesh-simplify：QEM 简化，生成 *_simplified.usd
3. export-glb：USD → GLB，生成交付格式
4. render-single：本地 Isaac Sim 出图，生成缩略图 / 合成数据
