Omniverse 平台与 USD backbone

从第一章开始，你已经把 USD 当作场景的数据模型来学习。这一节我们换个视角：站在 NVIDIA Omniverse 平台的角度看，为什么一个工业级仿真 / 协作系统会把 OpenUSD 选作唯一事实源（single source of truth），以及 Isaac Sim 如何建立在这个基础之上。

Omniverse 平台架构

Omniverse 是 NVIDIA 推出的实时 3D 协作与仿真平台，它的核心假设是：不同工具、不同团队、不同地理位置的协作者，共享同一个 USD 场景图。Omniverse 的组件栈可以分为三层：

最底层的 OpenUSD 核心 提供了场景图的数据模型——Stage、Prim、Layer、Composition arcs（第 1、2 章的内容）。中间的 Kit 框架把 USD 包裹成可插拔的扩展生态，暴露 Python API（omni.usd、carb.settings 等）。最上层是各种应用，Isaac Sim 就是其中一个专为机器人仿真设计的 Kit 应用。

Nucleus 协作服务 Omniverse 的多用户协同依赖 Nucleus 服务器，它像 Git 一样追踪 USD Layer 的修改，支持多客户端实时合并。每个编辑操作在 Layer Stack 上以 delta 形式广播，保证所有节点看到的是同一个 Composed Stage。

USD 作为唯一事实源

Omniverse 不使用自定义的场景格式，这是一个有意的设计决策。OpenUSD（Universal Scene Description）最初由 Pixar 为电影流水线开发，具备：

  • 非破坏性层叠（Non-destructive layering）：多个团队可以在不同 Layer 上 override 同一 Prim，最终由 LIVRPS 合成规则裁定。
  • Composition arcs：references、payloads、variants、clips——第 2 章你已经见过，这套机制让大场景可以按需加载和变体切换。
  • 跨工具互操作：Houdini、Maya、Blender、Unreal 都在 2020 年代陆续支持 OpenUSD 导入 / 导出。

对 Omniverse 来说，选择 USD 意味着不需要再发明一套私有场景格式——任何理解 USD 的工具都能直接接入 Nucleus 流。Isaac Sim 依托这一点，使得机器人仿真资产（几何、材质、物理属性）可以与设计工具无缝交换。

回顾：Stage / Prim / Layer 在第 1 章你学到，Stage 是"合成后的场景"，Layer 是写入磁盘的文件，Prim 是场景图里的节点。Omniverse 的每个视口本质上都在渲染一个 Stage，多用户协同就是多个 Layer 的实时 merge。

Isaac Sim 在 Omniverse 栈中的定位

Isaac Sim 是 NVIDIA 为机器人研究和开发设计的仿真平台，它在 Omniverse Kit 之上增加了两类关键能力：

  1. 物理仿真：集成 PhysX，提供刚体、关节、关节电机、接触力学等（下一节详讲）。
  2. 传感器仿真：相机（omni.isaac.sensor.Camera）、激光雷达（Lidar）、IMU 等，输出与真实传感器接口兼容的数据流。

Isaac Sim 的 Python 绑定（通过 pxr 库）让开发者既能用 USD API 直接操作场景图，也能调用 omni.isaac.core 的高层接口（如 World、RobotArticulation）。ConvertAsset 的所有工具（no-mdl 转换、mesh 简化、GLB 导出、render-single 出图）都依赖这同一套 Python 环境。

check_isaac_env.py
from pxr import Usd, UsdGeom          # OpenUSD 绑定，由 Isaac Sim 提供
from omni.isaac.core import World      # Isaac Sim 高层接口

# 确认 USD 和 Isaac 环境都正常
stage = Usd.Stage.CreateInMemory()
xform = UsdGeom.Xform.Define(stage, "/World")
print("USD ok:", xform.GetPrim().GetPath())

world = World()
print("Isaac Sim World ok:", world)
Isaac Sim Python 入口 ConvertAsset 的所有命令都通过 ./scripts/isaac_python.sh 调用，这个脚本负责设置 Isaac Sim 的 Python 环境（PYTHONPATH、LD_LIBRARY_PATH 等），然后执行 main.py。
