为什么需要 interchange format

在 GAMES101 里，你的渲染器只需要读一种格式——你自己写的。但一个电影工作室或机器人仿真平台的 pipeline 里，同一个资产要在十几个不同的应用里流转：建模器、材质编辑器、动画系统、灯光工具、渲染器、游戏引擎、仿真平台……每个应用都有自己偏好的内部格式。interchange format 的价值，正是在这些孤岛之间架桥。

From GAMES101 你在 GAMES101 用 OBJ 文件读取 mesh。OBJ 是一个简单的 interchange 格式：它的设计目标是「任何建模软件都能导出，任何渲染器都能导入」。USD 是这条思路的一个现代化、大规模化的延伸——但它不只传递 mesh，它传递整个场景描述，包括组合关系、材质、动画、灯光与物理属性。

DCC 孤岛问题

一个典型的 3D 内容生产 pipeline 里，数据需要在多个工具间流转。以电影制作为例：

  • 建模师在 Maya 或 Houdini 里做 mesh
  • 材质师在 Substance Painter 或 Mari 里刷贴图
  • 动画师在 Maya 里做动作
  • 灯光师在专用灯光软件里布光
  • 最终在 RenderMan 或 Arnold 里渲染

这些工具没有一个共同的 native 格式。历史上用 OBJ、FBX、Alembic 拼凑——每次导入导出都有信息损失，而且这些格式都不支持非破坏性编辑与层叠覆盖：灯光师想微调模型位置，必须在自己的副本上改，无法「在更强的层上覆盖建模师的 base」。

为什么 OBJ/FBX 不够 OBJ 只有 mesh 与简单材质，没有场景层次、没有引用、没有动画。FBX 是 Autodesk 私有格式，在场景引用与协作编辑上能力有限。Alembic 专注于 baked 动画缓存，不适合做资产组合的通用交换格式。

USD 的解法

Pixar 在 2016 年开源了 USD（Universal Scene Description），现在是皮克斯、工业光魔、NVIDIA 等公司的 pipeline 基础。USD 不只是一个文件格式，它是一套完整的 场景描述系统，提供：

  1. 统一数据模型：用 Prim（节点）、Attribute（属性）、Relationship（关系）描述任意 3D 数据。
  2. 层叠合成：多个 Layer 可以叠加，强层的意见覆盖弱层，支持多人协作在同一资产上各自工作而不冲突。
  3. 非破坏性覆盖：任何层都可以「覆盖」另一层的值，不需要复制整个文件。
  4. 可扩展 schema：几何（UsdGeom）、材质（UsdShade）、物理（UsdPhysics）都是插件化的 schema，可以扩展。
  5. 高性能运行时：Stage 是一个轻量级的合成 scenegraph，支持多线程访问，适合实时编辑。

USD 官方文档对它的定位是：「可以鲁棒且可扩展地交换任意 3D 场景的第一个公开可用软件。」

why_usd.usda
#usda 1.0
# 这是一个简单的 USDA 文本格式示例。
# 注意：整个场景描述是声明式的——描述"是什么"，而不是"怎么计算"。

def Xform "World" {
    def Mesh "Table" (
        # 这个 Prim 通过 reference 引用外部文件
        references = @./assets/table.usd@
    ) {
        # 可以在这里 override 外部文件的属性
        double3 xformOp:translate = (0, 0, 0)
    }

    def Mesh "Chair" (
        references = @./assets/chair.usd@
    ) {
        double3 xformOp:translate = (1.2, 0, 0)
    }
}

USD 在今天的地位

USD 现在已经是业界标准：

  • Pixar / DreamWorks / ILM：所有 3D 工作都在 USD 上进行。
  • NVIDIA Omniverse / Isaac Sim：用 USD 作为仿真场景的 native 格式。
  • 苹果 Vision Pro / Reality Composer：用 USDZ（ZIP 封装的 USD）作为 AR 内容格式。
  • Unreal Engine、Blender、Maya：均内置 USD 导入导出。

本书的 case study ConvertAsset 就是一个以 USD 为核心的工具链——它消费 USD（有 MDL 材质的 USD），生产 USD（有 UsdPreviewSurface 的 noMDL USD），最终导出为 GLB。理解 USD 是理解整个工具的前提。
