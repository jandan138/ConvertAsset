Layer 与 opinion

Layer 是 USD 的文件抽象——一个 Layer 对应一个物理文件（或内存中的匿名层），存储着若干 Prim 的若干属性值（以及 composition arc 的声明）。一个 Prim 的最终属性值，来自多个 Layer 里对它发表的 opinion 按强度规则合并的结果。这就是 USD 非破坏性编辑的底层机制。

From GAMES101 想象你在 Photoshop 里有多个图层：每一层可以覆盖下面的颜色，上面的层「更强」。USD Layer 的工作方式类似，区别是「被覆盖的对象」不是像素，而是「某个 Prim 的某个属性的值」。而且 USD 的层叠规则比 Photoshop 更复杂——它不只有简单的上下叠加，还有 reference / payload / variant 等多种 composition arc。

Layer：USD 的文件单元

每一个 .usd/.usda/.usdc 文件就是一个 SdfLayer（Sdf = Scene Description Foundation）。Layer 里存放的是原始数据：未经 composition 的 Prim 声明和属性值。

同一个 Stage 可能由数十个甚至数百个 Layer 组成：

  • 一个 root layer（你打开 Stage 时指定的文件）
  • root layer 通过 subLayers 引入的多个 sublayer
  • 各 Prim 通过 reference / payload 引入的外部 layer
scene.usda（root layer）
#usda 1.0
(
    # subLayers：在这个 root layer 之下叠加两个 layer
    # sublayer 比 root layer 弱（root 的 opinion 优先）
    subLayers = [
        @./lighting.usda@,    # 灯光师的层（较弱）
        @./animation.usda@    # 动画师的层（最弱）
    ]
)

# 在 root layer 里 override Table 的位置
over "World" {
    over "Table" {
        double3 xformOp:translate = (0, 0.5, 0)  # 这是一个 opinion
    }
}

Opinion：一个 Layer 对一个属性的声明

Opinion 是 USD 里最基本的数据单元：「某个 Layer 对某个 Prim 的某个属性，声明了某个值」。

一个属性可以在多个 Layer 里都有 opinion。Stage 的合成引擎按照 strength ordering（强度规则）决定哪个 opinion 生效：

  • 强度更高的 Layer 的 opinion，覆盖强度更低的。
  • 如果强度最高的 Layer 没有对某个属性发表 opinion，则用下一层的。
  • 如果没有任何 Layer 对该属性有 opinion，则使用 schema 的默认值（如果有的话）。
从 Photoshop 类比到 USD 的区别 Photoshop 的层覆盖是像素级的全图覆盖。USD opinion 的覆盖是属性级的细粒度覆盖：灯光师可以覆盖 /World/Table 的 visibility 属性，但不会影响同一个 Prim 的 points（顶点）属性——这两个属性来自不同的 Layer，各自按强度规则独立计算。

Layer Stack 与 composition

Layer Stack = root layer + 它的所有 subLayers（递归包含），按强度从强到弱排列。Stage 打开后，composition 引擎把所有 Layer 合并成一个统一的 scenegraph。

简单情况：两个 sublayer 和 root 组成一个 Layer Stack。

layer_stack_demo.py
from pxr import Usd, Sdf

# 创建三个 layer，模拟三人协作
base   = Sdf.Layer.CreateAnonymous(".usda")  # 建模师（最弱）
anim   = Sdf.Layer.CreateAnonymous(".usda")  # 动画师
shot   = Sdf.Layer.CreateAnonymous(".usda")  # 镜头（最强）

# 在 base layer 定义 mesh 及其颜色
base_spec = Sdf.PrimSpec(base, "Chair", Sdf.SpecifierDef, "Mesh")
col_attr = Sdf.AttributeSpec(base_spec, "primvars:displayColor",
                              Sdf.ValueTypeNames.Color3fArray)
col_attr.default = [(0.6, 0.4, 0.2)]  # 棕色

# 在 shot layer 覆盖颜色为红色（demo override）
shot_spec = Sdf.PrimSpec(shot, "Chair", Sdf.SpecifierOver)
over_attr = Sdf.AttributeSpec(shot_spec, "primvars:displayColor",
                               Sdf.ValueTypeNames.Color3fArray)
over_attr.default = [(0.9, 0.1, 0.1)]  # 红色（强层覆盖）

# 把三层叠成 Stage
shot.subLayerPaths = [anim.identifier, base.identifier]
stage = Usd.Stage.Open(shot)

prim = stage.GetPrimAtPath("/Chair")
# 最终合成值 = shot layer 的 opinion（最强）
attr = prim.GetAttribute("primvars:displayColor")
print(attr.Get())  # [(0.9, 0.1, 0.1)] — 红色胜出

ConvertAsset 里的 Layer 操作

ConvertAsset 的 no-MDL 转换核心操作正是对 Layer 的直接导出与改写：

processor.py（关键步骤）
# 关键：直接导出 root layer（不 flatten）
# GetRootLayer() 返回 SdfLayer，Export 写出物理文件
root_layer = stage.GetRootLayer()
root_layer.Export(dst_usd_abs)

# 然后打开新 stage，在新 layer 上改写路径与材质
dst_stage = Usd.Stage.Open(dst_usd_abs)
_rewrite_assets_in_stage(dst_stage, mapping)      # 改写 assetPath
convert_and_strip_mdl_in_this_file_only(dst_stage)  # 转换材质

# 再次导出，保存改写结果
dst_stage.GetRootLayer().Export(dst_usd_abs)

这里的关键是 GetRootLayer().Export()：它只导出当前 Layer 自己的数据，不会把 sublayer / reference 的内容内联进去。这就是「composition-preserving」的机制——composition arcs 的声明（references = @./chair.usd@）保留在新文件里，只是路径被改写为 @./chair_noMDL.usd@。
