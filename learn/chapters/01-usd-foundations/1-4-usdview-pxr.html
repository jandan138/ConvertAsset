usdview 与 pxr Python 速览

usdview 是 USD 官方自带的场景查看器与调试工具，也是理解 composition 结果的「真相窗口」——它直接调用同一套 Stage 合成引擎，你看到的就是代码里 stage.Traverse() 遍历到的结果。pxr 是 USD 的 Python 绑定，是 ConvertAsset 全部代码的核心依赖。这一节给出在实际工作中最高频的用法。

usdview：场景调试利器

usdview 随 Isaac Sim（和 USD 官方发行包）一起安装。启动方式：

terminal
# 在 Isaac Sim 环境里
usdview /path/to/scene.usd

# 或者通过 Isaac Sim python wrapper
./scripts/isaac_python.sh -c "
from pxr import Usd
stage = Usd.Stage.Open('/path/to/scene.usd')
for p in stage.Traverse():
    print(p.GetPath(), p.GetTypeName())
"

usdview 的三个最有用的面板：

  • Prim Browser（左侧树）：显示 composed Stage 里的 Prim 层次。点击 Prim 可以在右侧看它的所有 Properties 和每个 Property 上有哪些 opinion（来自哪个 Layer，值是什么）。
  • Layer Stack（菜单 View → Layer Stack）：显示当前 Stage 由哪些 Layer 组成，sublayer / reference 的嵌套关系一目了然。
  • Composition 面板（选中 Prim 后，右键 → Composition）：显示这个 Prim 的 composition arc 图——是被 subLayered 进来的，还是 referenced，还是 inherited？
ConvertAsset 调试时的最高频操作 转换完成后，用 usdview 打开 *_noMDL.usd，在 Layer Stack 面板验证「所有 sublayer / reference 路径是否已更新为 *_noMDL」；在 Prim Browser 里展开材质 Prim，确认 MDL shader 节点已被 UsdPreviewSurface 节点替换。

pxr Python 高频用法模式

以下是 ConvertAsset 代码里反复出现的 pxr 模式，也是你在 USD 工作中最常用的操作：

打开 Stage 与遍历

traverse.py
from pxr import Usd, UsdGeom, UsdShade

stage = Usd.Stage.Open("scene.usd")

# 遍历所有 Prim（深度优先，包含 inactive 的 Prim 不在此列表）
for prim in stage.Traverse():
    print(prim.GetPath(), prim.GetTypeName())

# 只遍历某个子树
world = stage.GetPrimAtPath("/World")
for prim in stage.Traverse(world.GetPath()):
    print(prim.GetPath())

# 遍历时过滤类型
for prim in stage.Traverse():
    if prim.IsA(UsdGeom.Mesh):
        mesh = UsdGeom.Mesh(prim)
        pts = mesh.GetPointsAttr().Get()
        print(f"{prim.GetPath()}: {len(pts)} points")

读写 Attribute

attributes.py
from pxr import Usd, UsdShade, Sdf, Gf

stage = Usd.Stage.Open("mat.usd")

# 找到 shader prim 并读取属性
shader_prim = stage.GetPrimAtPath("/Looks/Mat/PBRShader")
shader = UsdShade.Shader(shader_prim)

# 通过 schema API 读（推荐）
roughness_input = shader.GetInput("roughness")
if roughness_input:
    val = roughness_input.Get()
    print("roughness:", val)

# 通过底层 Attribute API 写
rough_attr = shader_prim.GetAttribute("inputs:roughness")
if rough_attr:
    rough_attr.Set(0.3)
else:
    # 如果属性不存在，先创建
    new_attr = shader_prim.CreateAttribute(
        "inputs:roughness", Sdf.ValueTypeNames.Float
    )
    new_attr.Set(0.3)

检查 composition arc 与 Layer Stack

composition_inspect.py
from pxr import Usd

stage = Usd.Stage.Open("scene.usd")

# 查看 root layer
root_layer = stage.GetRootLayer()
print("root layer:", root_layer.identifier)
print("sublayers:", root_layer.subLayerPaths)

# 查看所有 Layer（包括 referenced）
all_layers = stage.GetLayerStack()
for layer in all_layers:
    print("layer:", layer.identifier)

# 查看某个 Prim 的 composition arc query
prim = stage.GetPrimAtPath("/World/Chair")
query = Usd.PrimCompositionQuery(prim)
for arc in query.GetCompositionArcs():
    print("arc type:", arc.GetArcType(),
          "target:", arc.GetTargetNode().layerStack.identifier)

在 Stage 上创建/修改内容

author_stage.py
from pxr import Usd, UsdGeom, UsdShade, Sdf

# 创建新 Stage
stage = Usd.Stage.CreateNew("output.usda")

# 设置 defaultPrim（GLB 导出需要）
xform = UsdGeom.Xform.Define(stage, "/Root")
stage.SetDefaultPrim(xform.GetPrim())

# 设置 up 轴（Y-up for glTF，Z-up 是 USD 默认）
UsdGeom.SetStageUpAxis(stage, UsdGeom.Tokens.y)

# 保存
stage.GetRootLayer().Save()
# 或者导出到指定路径
stage.GetRootLayer().Export("/output/path/output.usda")

ConvertAsset 的 lazy import 模式

ConvertAsset 的所有模块都把 from pxr import ... 放在函数内部，而不是文件顶层。这是一个工程约定：

convert_asset/no_mdl/materials.py（模式示意）
def build_preview_surface(stage, material_path):
    # pxr 在函数体内导入——懒加载
    from pxr import UsdShade, Sdf  # type: ignore  # pxr provided at runtime

    mat = UsdShade.Material.Define(stage, material_path)
    shader = UsdShade.Shader.Define(stage, material_path + "/PBRShader")
    shader.CreateIdAttr("UsdPreviewSurface")
    # ...

这样设计的好处：CLI 的 --help 在没有 Isaac Sim 环境的机器上也能运行（不会因为 import pxr 失败而崩溃）；同时也方便测试时用 mock 替换 pxr。
