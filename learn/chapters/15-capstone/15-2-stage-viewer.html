在 viewer 里逐阶段看 asset

代码跑完了，但你怎么知道每一步是否真的做对了？这一节介绍用 usdview、inspect 子命令和浏览器端 GLB viewer 在流水线的每个阶段停下来观察资产，建立"眼见为实"的直觉。

阶段 1：原始 USD（MDL 材质）

用 usdview 打开原始 USD，在 Layer Stack 面板里可以看到材质 Layer 引用，在 Prim 属性面板里可以看到 info:id = "OmniPBR" 或其他 MDL 着色器 ID。

usdview 打开原始 USD
# usdview 由 Isaac Sim 安装提供
./scripts/isaac_python.sh -m usdview /data/assets/chair/chair.usd

在 usdview 中，展开 /Looks 下的材质 Prim，找到 Shader 节点，属性面板会显示 info:id = MDL 着色器名称（如 OmniPBR、OmniGlass），以及 inputs:* 参数。

同样可以用 inspect 子命令在命令行里检查 MDL 网络：

inspect MDL 材质
./scripts/isaac_python.sh ./main.py inspect \
    /data/assets/chair/chair.usd \
    mdl \
    /Looks/Material_Wood

阶段 2：noMDL USD（UsdPreviewSurface）

no-mdl 转换后，打开 *_noMDL.usd，同样的材质 Prim 现在应该显示 info:id = "UsdPreviewSurface"，并有 inputs:diffuseColor、inputs:metallic、inputs:roughness 等标准参数。

inspect noMDL 材质
./scripts/isaac_python.sh ./main.py inspect \
    /data/assets/chair/chair_noMDL.usd \
    usdpreview \
    /Looks/Material_Wood

# 预期输出（示例）：
# Material: /Looks/Material_Wood
#   Shader: UsdPreviewSurface
#   inputs:diffuseColor = (0.45, 0.32, 0.18)
#   inputs:metallic = 0.0
#   inputs:roughness = 0.65
#   inputs:normal = /Looks/Material_Wood/NormalMap (texture)

在 usdview 中并排打开 original 和 noMDL，视觉对比要点：

  • 高光分布：MDL 的 clearcoat 高光会消失，变成 Cook-Torrance roughness 高光。
  • 颜色保真：漫反射颜色应该大致一致（ConvertAsset 从 MDL 参数映射 diffuseColor）。
  • 纹理连接：贴图路径应该被重写为相对路径（指向 noMDL 目录）。

阶段 3：GLB — 用浏览器 viewer 查看

GLB 是 glTF 2.0 的二进制格式，可以直接在以下 viewer 中打开：

  • glTF Viewer（online）：gltf-viewer.donmccurdy.com，拖拽上传即可。
  • Khronos Sample Viewer：github.khronos.org，支持多种光照环境和 PBR 参数调试。
  • Blender：File → Import → glTF 2.0 / GLB，导入后在材质面板看 metallic-roughness 参数是否正确映射。

重点核查项：

  • 几何正确（face count 符合简化比例，no holes / flipped normals）。
  • 纹理正确（贴图内嵌在 GLB 里，texcoord 正确，UV 无撕裂）。
  • 材质参数合理（metallic 0 or 1，roughness 0–1，不出现纯黑 / 纯白异常）。
常见问题排查 材质全黑：inspect 检查 noMDL 的 diffuseColor 是否为 (0,0,0)，或检查 MDL 转换是否没有找到对应纹理。
UV 撕裂：face-varying UV 的 flattening（第 10 章）可能在某些边上产生接缝，用 mesh-simplify 的 cpp-uv backend 可以改善。
几何异常：usdview 里展开 Mesh Prim 查看 faceVertexCounts 是否全为 3（三角化），QEM 只支持纯三角网格。
