核心设计决策回顾

整本书的技术叙事围绕六个关键设计决策：

  1. 为什么选 USD 而不是私有格式？——互操作性和 Composition 机制（第 1、2 章）。非破坏性层叠让多工具协作成为可能，这正是 Omniverse 架构的基础。
  2. 为什么 MDL 需要转换？——renderer-bound 性质（第 5 章）使得 MDL 离开 RTX 就失效。UsdPreviewSurface 是移植性最好的 PBR 模型。
  3. 为什么 non-flattening（不展平）？——保留 references / variants 让转换可逆、可对比，不破坏原有工作流（第 6 章）。
  4. 为什么用 QEM 而不是随机简化？——QEM 的 quadric error metric 在数学上保证了误差最小化，同时支持 UV 保持（第 8 章）。
  5. 为什么 orbit camera 用 bbox 驱动而不是手动设置？——资产尺寸差异极大，自动取景才能批量化（第 13 章）。
  6. 为什么 evidence gate 要分四个独立门？——每个门回答不同维度的问题，不可替代；只有全部通过才能支持下游 robustness 主张（第 14 章）。

下一步去哪里

你现在掌握了从理论到工具的完整链条，有几个方向可以继续深入：

  • 扩展 ConvertAsset：为新的 MDL 着色器类型添加转换规则（no_mdl/materials.py），或支持更多 USD Schema（如 UsdPhysics、UsdLux）。
  • GRScenes 扩展 benchmark：把 pilot 阶段推进到 final benchmark（扩大 scene / VLM / 视角规模），更新 claims.yaml 中的 gate 状态。
  • Replicator 合成数据：用 Isaac Sim 的 Replicator 框架大规模生成训练数据，研究 noMDL 资产在 sim-to-real 场景下的迁移效果。
  • GPU 级优化：了解 CUDA kernel 的优化技巧（warp divergence、shared memory、Tensor Core 利用），将 mesh 处理和渲染流水线进一步加速。
