writer：拼出 GLB

glb/writer.py 里的 GlbWriter 不依赖任何第三方 glTF 库——它用 Python 内置的 struct 和 json 模块从零构建二进制文件。理解它的工作原理，等于深度理解了第 9 章 GLB 格式规范的所有细节。

From GAMES101 在 GAMES101 的光栅化 pipeline 里，framebuffer 是一块连续的字节数组。GLB 的 BIN 块与此类似：一块连续内存，里面依次存放顶点位置、法线、UV、索引、图片字节，通过 bufferView（偏移+长度）和 accessor（类型+count）来寻址。

GlbWriter 的内部状态

GlbWriter 维护两组数据：

  • JSON 结构列表：self.nodes、self.meshes、self.materials、self.accessors、self.buffer_views、self.textures、self.images、self.samplers、self.scenes
  • 二进制缓冲区：self.buffer_data（一个 bytearray），所有 numpy 数组和图片字节都追加到这里

添加 geometry 数据的管道

每次调用 add_mesh 时，内部会对 positions、normals、uvs、indices 各自调用 _add_accessor：

  1. 对齐：调用 _align_buffer(4)，用 \x00 填充到 4 字节对齐（glTF 规范要求）。
  2. 追加数据：把 numpy 数组的 .tobytes() 追加到 buffer_data。
  3. 创建 bufferView：记录 {buffer: 0, byteOffset, byteLength}。
  4. 创建 accessor：记录 {bufferView, componentType, count, type}，位置数据还需要 min / max 包围盒。
glb/writer.py — _add_accessor（精简）
def _add_accessor(self, data_np, component_type, count, type_str, ...):
    self._align_buffer(4)          # 4 字节对齐
    offset = len(self.buffer_data)
    self.buffer_data.extend(data_np.tobytes())

    bv = {"buffer": 0, "byteOffset": offset, "byteLength": len(data_np.tobytes())}
    self.buffer_views.append(bv)

    acc = {"bufferView": bv_idx, "componentType": component_type,
           "count": count, "type": type_str}
    self.accessors.append(acc)
    return acc_idx
