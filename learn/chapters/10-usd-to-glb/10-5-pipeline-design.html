cycle detection / dedup / lazy import 设计

ConvertAsset 的 USD 处理器（no_mdl/processor.py）需要递归地处理整棵 USD 资产树。生产级别的 USD 场景可能有上百个相互引用的 USD 文件，甚至出现循环引用。三个设计决策——cycle detection、dedup（去重）和 lazy import（懒加载）——共同保证了递归处理的安全、高效与正确。

From GAMES101 渲染器的 scene graph 遍历通常不需要考虑环——你不会让一个节点成为自己的祖先。但 USD 的 composition 允许跨文件引用，现实中的资产库可能出现 A 引用 B、B 又引用 A 的情况。处理这类图需要标准的图遍历技巧：visited 集合 + 栈内标记。

循环检测：in_stack + done

Processor 类维护两个状态：

  • self.in_stack: set[str] — 当前正在递归处理的 USD 文件绝对路径集合，用于检测环。
  • self.done: dict[str, str] — 已经处理完成的"源路径 → 目标 _noMDL 路径"映射，用于去重。

每次进入 process(src_usd_abs)：

  1. 如果 src_usd_abs in self.done：直接返回缓存结果（去重）。
  2. 如果 src_usd_abs in self.in_stack：检测到循环，打印警告并返回（避免无限递归）。
  3. 否则：加入 in_stack，递归处理子文件，处理完后加入 done，移出 in_stack。
no_mdl/processor.py — 循环检测与去重
def process(self, src_usd_abs):
    src_usd_abs = _to_posix(os.path.abspath(src_usd_abs))

    # 去重：已处理，直接复用结果
    if src_usd_abs in self.done:
        return self.done[src_usd_abs]

    # 环检测：正在处理自身的祖先，则循环
    if src_usd_abs in self.in_stack:
        print("[WARN] cyclic reference:", src_usd_abs)
        return self.done.get(src_usd_abs, src_usd_abs)

    self.in_stack.add(src_usd_abs)
    # … 递归处理子文件 …
    self.done[src_usd_abs] = dst_usd_abs
    self.in_stack.remove(src_usd_abs)
    return dst_usd_abs

这两个集合合在一起，构成了标准的 DFS with path tracking 算法：in_stack 是"当前路径"的标记，done 是"访问完成"的标记。

非扁平化（non-flattening）组合保持

ConvertAsset 的一个关键设计选择是：不 flatten。处理 USD 时，只复制当前文件的 root layer，不内联子文件的内容。这意味着：

  • references / payloads / sublayers / variants 的组合结构保持不变，只改写 assetPath 指向 *_noMDL 版本。
  • 子文件独立处理，主文件不会变得臃肿。
  • 处理结果可以被其他 USD 文件继续引用，保持了组合性（composability）。

懒加载（lazy import）

ConvertAsset 是作为一个 CLI 工具运行的，它的 Python 环境由 Isaac Sim 提供。pxr（USD Python 绑定）是一个极重的模块，启动时 import 它会增加数百毫秒的延迟，并且在没有 Isaac Sim 的环境下直接失败。

全仓库的做法是：只在函数体内部导入 pxr，不在模块顶层 import。例如：

典型的懒加载模式
# 错误方式：模块顶层 import（在无 pxr 环境下直接崩溃）
from pxr import Usd, UsdGeom   # ← 不应在顶层

# 正确方式：函数内 import（仅在真正需要时才加载）
def extract_mesh_data(usd_mesh):
    from pxr import UsdGeom     # ← 懒加载
    st_pv = UsdGeom.PrimvarsAPI(usd_mesh).GetPrimvar("st")
    ...

好处：

  • 可测试性：不依赖 pxr 的单元测试（如纯 Python 的数学函数）可以在不安装 Isaac Sim 的环境下运行。
  • 错误隔离：ImportError 发生在调用时，而不是模块加载时，方便定位问题。
  • 启动速度：CLI 的 --help 和参数验证不需要等待 pxr 加载。
