mesh 提取与 face-varying UV flattening

USD 允许纹理坐标（UV / st primvar）以"面变化"（face-varying）方式存储——同一个顶点在不同面上可以有不同的 UV 值。glTF 却只支持"每顶点一个 UV"（per-vertex UV）。填补这个语义鸿沟需要把 mesh 展平（vertex splitting）。这正是 glb/usd_mesh.py 里最核心的工作。

From GAMES101 GAMES101 里的 mesh 以"顶点数组 + 索引数组"的形式表达，每个顶点有唯一的位置和唯一的 UV。USD 为了节省存储，允许多个面角（face corner）共享同一个顶点位置，但拥有不同的 UV——这就是 face-varying。glTF 要求回到 GAMES101 的那种"展开"表示。

提取流程：四步

UsdMeshExtractor.extract_mesh_data(usd_mesh) 依次完成：

  1. 拓扑检查：读取 faceVertexCounts，若存在任何非三角形面则跳过该 mesh（GLB 要求三角化）。
  2. 提取 points：GetPointsAttr().Get()，转为 numpy (N, 3) float32。
  3. 提取 normals：仅处理顶点法线（数量与 points 相同），否则置 None。
  4. 提取 st primvar：检测 UV 的 interpolation（插值方式），决定是直接使用还是需要展平。

face-varying UV flattening 原理

USD 里 UV 的插值方式有三种主要情形：

  • vertex：每个顶点一个 UV，glTF 原生支持，直接传递。
  • faceVarying：每个"面角"一个 UV，即每个三角形的每个角（共 3×面数 个值）——glTF 不支持，必须展平。
  • uniform、constant：整面或全局一个值，相对少见。

展平算法（mesh flattening）：

代码中，展平逻辑如下：先把 points_np 按索引重排（points_np = points_np[indices_np]），使"新位置数组"的第 i 个元素对应面角 i 所用的空间点；再以同样方式重排法线和 UV；最后把 indices_np 替换为 arange(len(indices_np))——即简单的 0, 1, 2, … 序列，每个面角都变成了独立顶点。

glb/usd_mesh.py — face-varying 展平核心
if needs_flattening:
    # 新顶点数 = 所有面角数（3 × 面数）
    points_np = points_np[indices_np]      # 按面角展开位置
    if normals_np is not None:
        normals_np = normals_np[indices_np]  # 同步展开法线

    if uv_indices is not None and len(uv_indices) == len(indices_np):
        uvs_np = uv_data[uv_indices]       # 用 UV 索引映射
    else:
        uvs_np = uv_data                   # 直接对应面角

    # 新索引 = [0, 1, 2, 3, …]，每个面角独立
    indices_np = np.arange(len(indices_np), dtype=np.uint32)

代价与取舍

展平会增加顶点数。在极端情况（完全 face-varying、无共享 UV 的 mesh）下，顶点数会从 V 膨胀到 3F（F 为面数）。对于典型的有机模型，膨胀比通常在 1.5–2× 之间。这是为了获得 glTF 兼容性而付出的代价——好在顶点属性数据（position、normal、UV）本身已是 float32，后续 GlbWriter 会按 4 字节对齐写入，不会引入额外浪费。

知识点 面变化 UV 在 DCC 工具（Blender、Maya）里非常普遍，因为它允许把一个顶点的两条边缝开接缝（seam）。如果 ConvertAsset 不做这一步展平，导出的 GLB 在渲染器里看起来 UV 会错乱。
