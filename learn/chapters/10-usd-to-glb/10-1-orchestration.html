orchestration 总览

第 9 章讲清楚了 glTF 2.0 / GLB 的格式规范。这一章是 case study——我们直接读 ConvertAsset 仓库里 convert_asset/glb/ 的真实代码，看一个完整的 USD→GLB pipeline 是如何组织的。本节先纵观整体的 orchestration（协调调度）逻辑；后续四节分别深入各子模块。

From GAMES101 GAMES101 里的渲染器是一个把 scene → framebuffer 的单向流水线。production pipeline 则更像一次"数据格式翻译"：把 USD 的语义（Stage / Prim / Shader Graph）翻译成 glTF 的语义（node / mesh / accessor / material）。两者都是对同一个 3D 世界的不同"语言描述"。

入口：UsdToGlbConverter.process_stage

glb/converter.py 里的 UsdToGlbConverter 是整个管线的主控。它的 process_stage(src_usd_path, out_glb_path) 方法按顺序完成以下五步：

  1. 打开 Stage：用 Usd.Stage.Open 加载 USD 文件。
  2. 坐标系对齐：检测 up axis；若为 Z-up，构造一个绕 X 轴旋转 −90° 的根矩阵，使输出与 glTF 的 Y-up 约定对齐。
  3. 场景图构建：调用 UsdSceneGraphBuilder.build（usd_scene.py）遍历 Stage，返回一组按拓扑排序的 SceneNodeDesc。
  4. 逐 Mesh 转换：对每个 is_mesh=True 的节点，调用内部 _convert_mesh，依次提取几何数据和材质数据，再通过 GlbWriter 写出。
  5. 写出 GLB：调用 self.writer.write(out_glb_path) 生成最终二进制文件。
glb/converter.py（精简版）
class UsdToGlbConverter:
    def __init__(self):
        self.writer = GlbWriter()
        self.root_transform = Gf.Matrix4d(1.0)
        self._image_cache = {}          # 防止同一纹理重复写入 BIN 块

    def process_stage(self, src_usd_path, out_glb_path):
        # 每次调用重置状态，避免多次复用时串场
        self.writer = GlbWriter()
        self._image_cache = {}

        stage = Usd.Stage.Open(src_usd_path)

        # Z-up → Y-up 坐标系旋转
        if UsdGeom.GetStageUpAxis(stage) == 'Z':
            rot = Gf.Rotation(Gf.Vec3d(1, 0, 0), -90)
            self.root_transform = Gf.Matrix4d(rot, Gf.Vec3d(0))

        # 场景图构建 → 按拓扑顺序排列所有节点
        scene_nodes = UsdSceneGraphBuilder.build(stage, root_transform=self.root_transform)

        # 两遍处理节点：第一遍添加 mesh / node；第二遍绑定父子关系
        node_indices = {}
        for desc in scene_nodes:
            mesh_index = self._convert_mesh(...) if desc.is_mesh else None
            node_indices[desc.key] = self.writer.add_node(...)

        for desc in scene_nodes:
            # 将节点挂到父节点或 scene root
            ...

        self.writer.write(out_glb_path)

状态隔离设计

注意 process_stage 第一步就重置了 self.writer 和 self._image_cache。这是一个防御性设计：同一个 UsdToGlbConverter 实例可以被顺序调用多次处理不同 USD 文件，而不会因为上次残留数据产生污染。

_image_cache 是一个 dict[str, int]，键是纹理文件路径或组合键（如 MR_<metal>_<rough>），值是 GlbWriter 里的 image 索引。同一 USD 场景中多个 mesh 共享同一张 BaseColor 贴图时，只会把字节写入 BIN 块一次。
