material / texture 提取

USD 用 Shader Graph（着色器节点网络）表达材质；glTF 用 pbrMetallicRoughness JSON 对象加纹理引用表达材质。两套系统的语义几乎一一对应，但 glTF 有一个独特要求：metallic 和 roughness 必须打包到同一张图片的不同通道（channel packing）。

From GAMES101 GAMES101 里你写的 shader 直接在代码里 sample(texture, uv)。USD 把这个"代码"变成了数据：一张 Shader Graph，节点之间用连线（connection）表达数据流。提取材质就是遍历这张图，找到 UsdPreviewSurface 节点和它连接的 UsdUVTexture 节点。

三步查找材质

UsdMaterialExtractor.extract_material_data(usd_mesh) 的工作分三步：

  1. 查找绑定：用 UsdShade.MaterialBindingAPI(usd_mesh).ComputeBoundMaterial() 找到与该 Mesh Prim 绑定的 UsdShade.Material。
  2. 定位着色器：在 Material Prim 下遍历所有子 Prim，找到 info:id = "UsdPreviewSurface" 的 Shader Prim。
  3. 提取输入：读取 diffuseColor、roughness、metallic 的数值；对每个输入追踪连线（GetConnectedSource），若连接到 UsdUVTexture 节点则读取其 inputs:file 属性获取贴图路径。
glb/usd_material.py — 查找 UsdPreviewSurface
for child in Usd.PrimRange(mat.GetPrim()):
    if child.GetTypeName() == "Shader":
        sh = UsdShade.Shader(child)
        if sh.GetIdAttr().Get() == "UsdPreviewSurface":
            shader = sh
            break

贴图路径解析

USD 里的纹理路径存在 inputs:file 属性中，类型是 Sdf.AssetPath，其 .path 字段存储的可能是相对路径。代码在拿到路径后，会检查是否为绝对路径；如不是，则相对于当前 Stage 的 root layer 目录解析为绝对路径，确保在写入 GLB 时能读到文件字节。

Metallic-Roughness Channel Packing

glTF 规范要求 metallicRoughnessTexture 的通道布局如下：

  • B 通道（蓝）：metallic
  • G 通道（绿）：roughness
  • R 通道：未使用（通常为 0 或 1）

texture_utils.py 里的 pack_metallic_roughness(metal_path, rough_path) 函数负责把两张独立的灰度图读入，将 metallic 写入蓝通道、roughness 写入绿通道，打包成一张 RGB PNG，返回字节流。这个过程在 converter.py 里用组合键缓存（key = f"MR_{metal_path}_{rough_path}"），避免同场景多 mesh 重复打包。

默认值与容错

当 USD 材质没有设置某个参数时（如没有 diffuseColor 输入），代码给出合理的默认值：

  • diffuseColor → (1.0, 1.0, 1.0, 1.0)（白色、不透明）
  • roughness → 0.5
  • metallic → 0.0

这样即使输入 USD 材质不完整，导出的 GLB 也能正常显示（以默认 PBR 参数渲染），而不会崩溃。
