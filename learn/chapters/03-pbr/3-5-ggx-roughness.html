GGX (Trowbridge-Reitz) 与 roughness

GGX（即 Trowbridge-Reitz 分布）是当今实时和离线渲染里最主流的 NDF（normal distribution function）。它的核心优势在于「长尾」（long tail）——比 Beckmann 等经典分布更好地拟合真实材质数据，尤其是金属的发光高光拖尾。同样重要的是 roughness² 重映射：让艺术家调整 roughness 滑块时，视觉变化更「均匀」。

From GAMES101 上一节（3.4）给出了 GGX NDF 的完整公式 \(D_\text{GGX}(h) = \alpha^2 / [\pi((n\cdot h)^2(\alpha^2-1)+1)^2]\)。这里我们深入理解它的形状、与 Beckmann 的对比，以及为什么要做 \(\alpha = \text{roughness}^2\) 这一步重映射。

GGX 的形状：为什么叫「长尾」

NDF 描述了微表面法线 \(m\) 的概率密度——法线朝向越接近 half vector \(h\)，高光越亮。不同 NDF 的区别主要体现在高光「峰」的锐度和远离峰值后的「尾部」衰减速率。

对比三种常见 NDF：

  • Phong / Blinn-Phong NDF：用 \((n\cdot h)^p\) 近似，高光峰值锐，尾部指数衰减，衰减极快。
  • Beckmann NDF：基于高斯分布的坡度统计，衰减比 Phong 慢，但仍不够真实。
  • GGX（Trowbridge-Reitz）：分母是 \(((n\cdot h)^2(\alpha^2-1)+1)^2\) 的平方，衰减最慢，产生明显的「发光拖尾」。

Disney 对 MERL 100 真实材质数据库的拟合研究（2012）发现：几乎所有真实材质（尤其是抛光金属）的高光尾部都比 Beckmann 更宽。GGX 能更好地匹配这些数据，这是它取代 Beckmann 成为 UE4、Unity、glTF、UsdPreviewSurface 默认选择的根本原因。

GGX NDF 公式详解

$$ D_\text{GGX}(h) = \frac{\alpha^2}{\pi\,\bigl((n\cdot h)^2\,(\alpha^2 - 1) + 1\bigr)^2} $$

其中 \(\alpha\) 是 GGX 的内部粗糙度参数（不是直接暴露给艺术家的 roughness）。当 \(\alpha \to 0\)：分布收缩为 Dirac delta，表面完全光滑（镜面）；当 \(\alpha \to 1\)：分布完全均匀，表面完全漫射。

关键性质：NDF 必须在半球上归一化，即 \(\int_\Omega D(h)\cos\theta_h\,d\omega = 1\)。GGX 有解析形式的归一化常数（分母里的 \(\pi\)），这使得它既物理正确又易于高效实现。

roughness² 重映射：感知线性

PBR 管线（UsdPreviewSurface、glTF、Disney BRDF）暴露给艺术家的参数是 roughness \(\in [0, 1]\)，但喂进 GGX 公式的是 \(\alpha\)。标准做法是：

$$ \alpha = \text{roughness}^2 $$

为什么不直接用 \(\alpha = \text{roughness}\)？直觉如下：

  • 当 roughness 从 0.0 线性增加到 1.0 时，视觉上大多数有趣的变化集中在低端（0.0 → 0.3 之间高光快速扩散）。如果直接令 \(\alpha = \text{roughness}\)，滑块低端区间占据了大量「视觉变化空间」，中高端则变化平缓，不直觉。
  • 平方映射把这个感知曲线「拉直」了：在整个 \([0,1]\) 区间内，roughness 滑块移动相同距离对应大致相同的视觉变化量。
  • Disney 在 2012 年 BRDF 研究中明确推荐此映射，称其使「roughness 从粗糙到光滑的插值更为直觉」。
实现时的常见坑 许多 shader 代码和 UsdPreviewSurface 规范都强调：roughness 在进入 D 项和 G 项之前必须先平方得到 \(\alpha\)，然后才能代入 GGX 公式。忘记这步会让低 roughness 的材质看起来比预期粗糙得多。

Smith G 与 roughness 的关系

geometry 项 G 同样依赖 \(\alpha\)。Smith 方法将 shadowing 和 masking 分别处理，再相乘：

$$ G(l, v, h) = G_\text{sub}(n, v, k) \cdot G_\text{sub}(n, l, k) $$

其中 Schlick-GGX 近似：

$$ G_\text{sub}(n, x, k) = \frac{n\cdot x}{(n\cdot x)(1-k) + k}, \qquad k_\text{direct} = \frac{(\alpha+1)^2}{8} $$

注意这里的 \(k\) 用的是直接光版本，基于 \(\alpha = \text{roughness}^2\)。当 roughness = 0 时 \(k \approx 1/8\)，G 接近 1（几乎无遮挡）；当 roughness = 1 时 \(k = 0.5\)，G 明显小于 1（严重遮挡）——这正确地模拟了粗糙表面微结构导致的能量损失。

在 UsdShade 中指定 roughness

ConvertAsset 的 no-MDL 转换流程把材质落到 UsdPreviewSurface。roughness 参数直接对应 MDL 中的粗糙度输入，转换时按照 spec 规定不做额外的平方——平方由 renderer 在 shader 内部完成。

set_roughness.py
from pxr import UsdShade, Sdf

# roughness 直接设为艺术家友好的感知线性值 [0,1]
# renderer 内部会做 alpha = roughness^2 再喂进 GGX
shader.CreateInput("roughness", Sdf.ValueTypeNames.Float).Set(0.35)

# 低 roughness → 接近镜面（sharp highlight）
# 高 roughness → 漫射高光（wide, dim highlight）
shader.CreateInput("roughness", Sdf.ValueTypeNames.Float).Set(0.85)
