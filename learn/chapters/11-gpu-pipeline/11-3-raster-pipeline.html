rasterization pipeline 在硬件上

上一节讲了 SM 的内部结构。这一节从外部视角看：当一帧图像被渲染时，三角形数据是怎样流过 GPU 的各级硬件单元，最终变成像素写入 framebuffer 的？

From GAMES101 GAMES101 里"逐三角形遍历 → bounding box 内逐像素判断"的软件实现，在 GPU 里被分解成多个专用阶段：三角形经过 Vertex Shader（SM）→ PolyMorph Engine（三角形建立）→ Raster Engine（覆盖测试）→ Fragment Shader（SM）→ ROP（z-test + blend）。

GPC / TPC 层级

Turing TU102 的硬件层级：

  • GPC（Graphics Processing Cluster）：最顶层的调度单元。每个 GPC 有一个专用的 Raster Engine 和 6 个 TPC。TU102 共 6 个 GPC。
  • TPC（Texture Processing Cluster）：包含 2 个 SM 和纹理单元（Texture Unit）。TU102 共 36 个 TPC。
  • SM：运行 vertex / fragment / compute shader 的基本单元。

顶点处理 → 光栅化

一帧的处理流程：

  1. 顶点分发：驱动程序把顶点 buffer 切分成小批次，分发给各 SM 上的 Vertex Shader warp。每个顶点执行一次 vertex shader，输出裁剪空间坐标和插值属性（法线、UV 等）。
  2. 图元汇集 + 裁剪：PolyMorph Engine 收集 vertex shader 输出，拼成三角形，执行视锥体裁剪（frustum clipping）和背面剔除（back-face culling）。
  3. 光栅化：每个 GPC 内的 Raster Engine 接收三角形，执行扫描转换（scan conversion），生成覆盖的像素列表和重心坐标插值结果。这是一个固定功能单元，专为速度优化，每个 GPC 每帧可处理数亿像素。
  4. Fragment Shader：覆盖的像素（称为 fragment）被分批分发给各 SM。每个 fragment 执行一次 fragment shader，读取材质、计算光照，输出颜色值。
  5. ROP（Render Output Processor）：执行最终的 z-buffer 测试（depth test）和 alpha blending。TU102 有 96 个 ROP，均匀分布在内存控制器旁边，以最大化 framebuffer 写入带宽。

纹理单元

Texture 采样在 fragment shader 里频繁发生，因此 TPC 里专门有 Texture Unit（TMU，Texture Mapping Unit）做三线性过滤（trilinear filtering）和各向异性过滤（anisotropic filtering）。TU102 有 288 个 TMU，总纹素填充率（texel fill rate）为 444 Gigatexels/sec（Founders Edition，1635 MHz）。

瓶颈分析

一帧的性能取决于哪个阶段先饱和：

  • vertex-bound：顶点数极多、顶点 shader 复杂。Mesh Shading（Turing 新特性）可以减轻 CPU 侧的 draw call 瓶颈。
  • fragment-bound（最常见）：屏幕面积大、fragment shader 复杂或采样纹理多。
  • memory-bound：纹理或 framebuffer 数据超出缓存，带宽成为瓶颈。GDDR6 在 TU102 上提供 616 GB/s。
  • ROP-bound：高分辨率 + 大量 alpha blending。
小结 rasterization pipeline 在 GPU 上是一条可编程（SM，vertex / fragment shader）与固定功能（Raster Engine、ROP）硬件混合的流水线。固定功能阶段做的事情是每帧固定的，用专用电路可以实现极高吞吐；可编程阶段是 SIMT 大规模并行，数千个 fragment 同时着色。这套组合在过去 20 年里驱动了游戏图形质量的持续提升——但它有一个根本性的局限：不适合 ray tracing。
