RT Core：traversal + ray-triangle intersection

Turing 的 RT Core 是世界上第一个专用的实时光线追踪硬件加速器。每个 SM 内嵌一个 RT Core，它接收 SM 发出的"光线探针"，自主完成全部 BVH traversal 与 ray-triangle intersection 工作，然后向 SM 返回命中或未命中的结果。SM 在 RT Core 工作期间可以同时执行其他 shader 任务。

From GAMES101 GAMES101 里的 ray tracing 代码需要递归地遍历 BVH，调用 ray-box 和 ray-triangle 测试函数。RT Core 把这整段代码变成了一个硬件状态机：SM 只需"启动一条光线"，RT Core 负责所有的 BVH 遍历循环和几何测试，最后返回结果。

RT Core 内部：两个专用单元

Turing 白皮书明确描述 RT Core 包含两个专用子单元：

  • Box-test unit（包围盒测试单元）：专门执行 ray-AABB intersection——即判断光线是否与轴对齐包围盒相交。这在 BVH 的每一层都要调用，是遍历中最频繁的操作。用硬件实现可以在数个时钟周期内完成，比 SM 用 shader 代码做要快数十倍，且不占用 CUDA core 的指令槽。
  • Triangle-test unit（三角形测试单元）：在到达叶节点后，执行精确的 ray-triangle intersection（通常基于 Möller–Trumbore 算法）。它接收叶节点内的三角形数据，返回 hit 与否、重心坐标和命中距离 $t$。

卸载 SM 的收益

RT Core 完成 BVH 遍历时，SM 不需要执行任何指令——RT Core 是完全异步的。这有两重好处：

  1. 释放 SM 指令槽：每条光线不再消耗数千条 SM 指令，SM 可以用这些算力做 vertex / fragment / compute shading。
  2. 消除 SIMT 分歧惩罚：不同光线的 BVH 遍历路径分歧（divergence）现在在 RT Core 内部处理，不会拖累整个 warp。

性能数据（来自 Turing 白皮书）

Turing 白皮书 Appendix D 和 RT Core 章节记录：

  • Pascal（无 RT Core）：约 1.1 Giga Rays/sec（纯软件 BVH 遍历，消耗约 10 TFLOPS 算力）。
  • Turing TU102 RTX（有 RT Core）：10 Giga Rays/sec（RT Core 加速，约 10× 提升）。

这 10× 提升的来源：一是 box-test 和 triangle-test 的硬件加速本身，二是 SM 不再被 BVH 遍历占用，可以处理更多 shading 工作，整体吞吐大幅提升。

API 侧 应用开发者通过 DXR（DirectX Raytracing）、NVIDIA OptiX、或 Vulkan ray tracing API 调用 RT Core。API 使用 TraceRay()（DXR）/ rtTrace()（OptiX）等调用；驱动程序负责 BVH 构建和数据布局，RT Core 负责执行遍历。
