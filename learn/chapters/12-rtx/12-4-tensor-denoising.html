Tensor Core 与 denoising / DLSS

RT Core 解决了 BVH 遍历太慢的问题；但即使有 RT Core，实时帧预算内每像素只能发射少量光线（通常 1–4 条），结果图像充满噪点（noise）。Tensor Core 通过 AI denoising 把稀疏噪点图像恢复成清晰图像；DLSS（Deep Learning Super-Sampling）则用 AI 超分把低分辨率渲染结果放大到输出分辨率，进一步节省算力。

From GAMES101 GAMES101 里 path tracing 每像素需要数百至数千 spp（samples per pixel）才能收敛，消除噪点。实时 ray tracing 每像素只有 1–4 spp——这在数学上是欠采样（undersampling）。AI denoising 用神经网络的"先验知识"在欠采样图像上重建高质量结果，本质上是一个空间重建（spatial reconstruction）问题。

Tensor Core 结构

Tensor Core 是专门为矩阵乘法（GEMM）优化的执行单元，是深度学习训练和推理的关键算力来源。Turing SM 每个处理块有 2 个 Tensor Core，每个 SM 共 8 个，TU102 全 GPU 共 576 个 Tensor Core。

Turing Tensor Core 支持 FP16 矩阵运算（with FP16 or FP32 accumulation）以及专为推理优化的 INT8 和 INT4 精度。峰值性能（RTX 2080 Ti Founders Edition）：

  • FP16 Tensor TFLOPS with FP16 accumulate：113.8 TFLOPS
  • INT8 Tensor TOPS：227.7 TOPS
  • INT4 Tensor TOPS：455.4 TOPS

与同代 CUDA core（14.2 TFLOPS FP32）相比，Tensor Core 在矩阵操作上的吞吐是普通算力的约 8×——但只能做矩阵乘法这一种操作。

AI Denoising

每帧 ray tracing 只生成 1 spp 的图像（噪点极多）。AI denoising 流程：

  1. 输入：当前帧的噪点 ray tracing 结果 + G-buffer（法线、深度、材质）
  2. 在 Tensor Core 上运行一个轻量 CNN（卷积神经网络），识别噪点并重建干净图像
  3. 可选：利用上一帧结果做时域累积（temporal accumulation），进一步提升稳定性

效果：1 spp 的 ray tracing 图像经 AI denoising 后，质量接近 16–64 spp 的结果。这使得实时帧预算内（16 ms at 60 fps）可以得到视觉上可接受的 ray traced 效果。

DLSS（Deep Learning Super-Sampling）

DLSS 是 Tensor Core 的另一个关键应用。其工作流程：

  1. 以低于目标分辨率渲染（例如 1440p 目标，但以 960p 渲染）
  2. 使用一个在 64x 超采样"地面真值"图像上训练的深度神经网络，将低分辨率帧结合运动向量和历史帧推断出高分辨率结果
  3. 输出质量接近目标分辨率的 TAA（Temporal Anti-Aliasing）效果，但仅用约一半的 shading 工作

Turing 白皮书数据（Figure 22）：RTX 2080 Ti + 4K DLSS 的帧率是 GTX 1080 Ti（Pascal）+ 4K TAA 的约 2×。这意味着不仅 ray tracing 得到了加速，整体渲染性能也因 DLSS 大幅提升。

三类核心协作

Turing RTX 的核心价值在于三类处理单元的协作：

  • CUDA Core：负责 vertex / fragment shading（可编程、通用）
  • RT Core：负责 BVH traversal + ray-triangle intersection（固定功能、专用）
  • Tensor Core：负责 AI denoising + DLSS（矩阵运算专用）

三者并发执行，互不干扰，使整帧渲染既快速（rasterization）又高质量（ray tracing）又清晰（AI denoising）。
