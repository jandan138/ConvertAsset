rasterization vs RT 混合管线

即使有了 RT Core，完全用 ray tracing 渲染整帧在实时场景仍然太贵。NVIDIA 的答案是 Hybrid Rendering（混合渲染）：用 rasterization 做快速的主可见性判断，用 ray tracing 生成高质量的反射、折射、阴影、ambient occlusion——各用所长。

From GAMES101 GAMES101 里 Whitted-style ray tracing 是从主光线（primary ray）开始递归地发射反射/折射二级光线。混合管线把"主光线"这一步替换成了 rasterization（z-buffer 确定可见性），然后只对需要高质量效果的像素发射二级光线（secondary ray）。这样大幅减少了 ray tracing 的工作量。

混合管线逻辑

Turing 白皮书（Figure 16 / 17）描述的混合管线：

  1. G-buffer pass（光栅化）：用标准 rasterization 渲染整帧，生成每像素的深度、法线、材质参数（G-buffer）。这一步速度极快，确定了所有可见几何。
  2. 二级光线（Ray Tracing）：
    • Reflections（反射光线）：从 G-buffer 读取法线，发射反射光线，命中点的颜色通过 RT Core 确定。
    • Shadows（阴影光线）：从可见点向光源发射阴影光线，判断遮挡。
    • Ambient Occlusion（环境遮蔽光线）：短距离随机光线，判断局部遮挡。
  3. 合成（Composition）：将光栅化结果与 ray tracing 效果合并，输出最终帧。往往还需要 denoising（第 12-4 节讲）去除稀疏采样带来的噪点。

什么时候用 ray tracing 收益最大

Turing 白皮书总结了几类最适合 ray tracing 的效果：

  • 反射（Reflections）：rasterization 的屏幕空间反射（SSR）只能反射屏幕上可见的物体；ray tracing 反射可以反射任何场景几何。
  • 阴影（Shadows）：Shadow map 有分辨率限制和漏光问题；ray traced shadows 天然精确，可做柔和面光源阴影。
  • 环境遮蔽（AO）：屏幕空间 AO（SSAO）只看到屏幕内的几何；ray traced AO 基于真实场景几何。
  • 全局光照（Global Illumination）：光栅化无法做到，需要 path tracing 或 RT 辅助计算。

API 支持：DXR / OptiX / Vulkan

Turing 的 RT Core 通过以下 API 暴露：

  • DXR（DirectX Raytracing）：Microsoft 发布于 2018 年初，与 Turing 同期推出。
  • NVIDIA OptiX：NVIDIA 自家的光线追踪 SDK，用于离线渲染和专业应用。
  • Vulkan ray tracing：Khronos 的跨平台标准。
接下来 12-4 讲另一个关键模块：Tensor Core 怎样驱动 denoising 和 DLSS，让稀疏采样的 ray tracing 结果变成视觉上清洁的图像。
