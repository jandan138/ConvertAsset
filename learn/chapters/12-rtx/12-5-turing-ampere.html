Turing → Ampere 演进（whitepaper 数据）

NVIDIA Ampere GA10x（2020 年，RTX 30 系列）是继 Turing 之后的下一代 GPU 架构。Ampere 在三个关键维度上进化：2× FP32 吞吐、第二代 RT Core（ray-triangle 测试吞吐翻倍）、以及新的 RT + compute 并发执行能力。本节直接引用 NVIDIA Ampere GA102 白皮书的数据，与 Turing 进行对比。

From GAMES101 从 Turing 到 Ampere，RT Core 的进化类比于 GAMES101 里的 BVH 优化：Turing 第一代 RT Core 建立了硬件加速的基础（第一次在 GPU 里加了专用 BVH 遍历单元）；Ampere 第二代 RT Core 主要提升了 ray-triangle intersection 测试的吞吐量，并让 RT Core 和 compute 能真正并发运行，就像优化的 BVH 不仅降低了每条光线的遍历开销，还让遍历和着色可以真正重叠。

Ampere GA10x SM：2× FP32

Turing SM 的每个处理块（partition）有两条数据通路，但只有一条能处理 FP32；另一条仅限 INT32。Ampere 把两条通路都改为支持 FP32，FP32 峰值吞吐直接翻倍。

  • 每个 Ampere SM partition：可同时执行 32 FP32 操作/周期（vs Turing 的 16）
  • 每个 Ampere SM：128 CUDA Core（vs Turing 的 64）
  • GA102（RTX 3090）：10496 CUDA Core，>35 TFLOPS FP32（vs Turing TU102 4608 CUDA Core，14.2 TFLOPS）

Ampere 白皮书（Table 1）：相对于 Pascal（GP102），Turing FP32 是 1×，Ampere FP32 是 2×（在等效时钟频率下）。这使 ray tracing denoising shader 等 FP32 密集型工作负载大幅受益。

第二代 RT Core

Ampere GA10x 白皮书中对第二代 RT Core 的描述：

"Like Turing, the second-generation RT Core in GA10x includes dedicated hardware units for BVH traversal and ray-triangle intersection testing… Ampere architecture RT Cores double the ray/triangle intersection testing rate over Turing architecture RT Cores."

关键点：

  • ray-triangle intersection 吞吐翻倍（~2× over Turing）
  • BVH traversal 和 triangle test 的基本结构与 Turing 相同，仍是 box-test unit + triangle-test unit
  • 改善了缓存子系统，与 GA10x SM 的 128 KB L1（Turing 96 KB）配合，减少 BVH 遍历时的内存访问延迟

整体 ray tracing 性能（Ampere 白皮书，Figure 5）：RTX 3080 在 ray tracing workload 上比 RTX 2080 Super 快约 2×。

RT Core + Compute 并发执行

Turing 支持 "Async Compute"（compute 与 graphics 并发），但 RT Core 操作不能与其他 compute 工作完全并发。Ampere 的一项重要升级：

"The GA10x GPU enhances the Async Compute functionality… with a new capability that allows RT Core and graphics, or RT Core and compute workloads to be processed concurrently in each GA10x GPU SM."

实际意义：在 Ampere 上，一个 SM 可以同时跑 RT Core 的 BVH 遍历 和 compute shader（例如 AI denoising）。这消除了 Turing 上"RT Core 工作时 SM 只能等待或做不相关的图形工作"的限制，进一步提高了硬件利用率。

第三代 Tensor Core + Sparsity

Ampere 还升级了 Tensor Core：

  • 支持 TF32（TensorFloat-32）精度，AI 训练吞吐比 Turing FP32 快 5×
  • 新增 Sparsity（稀疏性）支持：对稀疏神经网络，可以跳过零权重计算，Tensor TFLOPS 再翻倍（sparse 模式）
  • DLSS 升级到 DLSS 8K，支持 8K 分辨率游戏

Turing vs Ampere 关键数字对比

Chapter 12 小结 RTX 架构的本质是三种硬件核心的协作：CUDA Core（通用 shader）、RT Core（BVH traversal + ray-triangle intersection 的固定功能卸载器）、Tensor Core（AI denoising + DLSS 的矩阵运算引擎）。Turing 是第一代实现，把软件 ray tracing 的速度提升 10×；Ampere 是第二代进化，triangle-test 吞吐翻倍、FP32 算力翻倍、RT + compute 真正并发，整体 ray tracing 速度再提升约 2×。这套架构已成为 NVIDIA GPU 的标准设计，持续演进至 Ada Lovelace（RTX 40 系列）和 Blackwell（RTX 50 系列）。
