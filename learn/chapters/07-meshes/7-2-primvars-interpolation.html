primvars 与 interpolation

primvar（primitive variable）是 USD 附着在几何 prim 上的"每元素数据"——UV、法线、顶点色、蒙皮权重……都可以是 primvar。USD 提供五种 interpolation 模式，描述这些数据如何在面的内部插值。搞清楚插值模式，是理解后续 face-varying 陷阱与 GLB 导出时"顶点分裂"的关键前提。

From GAMES101 你在光栅化作业里用过重心坐标（barycentric coordinates）在三角形内插 UV 与法线——这正是 vertex 插值的含义：数据定义在顶点上，三角形内部用重心坐标线性插值。USD 的 primvar 系统让你可以为不同数据选择不同的插值粒度。

五种插值模式

从粗到细，五种 interpolation 模式如下：

模式 每 prim 几个值 典型用途
constant 1 整个 prim 共享一个值（如对象 ID）
uniform 面数 每面一个值（如面材质标记）
varying 顶点数（线性插值） 曲面上的线性插值量（细分前用）
vertex 顶点数 顶点 UV、顶点色（glTF 原生支持）
faceVarying 面角点数（∑ faceVertexCounts） UV 缝合线、硬边法线（最常用，也最棘手）

对三角形网格，varying 与 vertex 行为相同；在细分曲面中两者有差异。实践里你几乎只会遇到 vertex 和 faceVarying。

用 PrimvarsAPI 读取

通过 UsdGeom.PrimvarsAPI 访问 primvar，可以同时拿到数据、插值模式和可选的 indices（压缩存储用）：

read_primvar.py
from pxr import UsdGeom

api  = UsdGeom.PrimvarsAPI(mesh)
st   = api.GetPrimvar("st")        # UV 坐标，通常命名为 "st"

print(st.GetInterpolation())       # "faceVarying" 或 "vertex" 等
print(len(st.Get()))               # 数据条目数
print(st.GetIndices())             # 若有压缩索引则非空

数据条目数规则

根据插值模式，primvar 数据的条目数有严格约束：

$$ \text{条目数} = \begin{cases} 1 & \text{constant} \\ |\text{faces}| & \text{uniform} \\ |\text{points}| & \text{varying / vertex} \\ \sum_i \text{faceVertexCounts}[i] & \text{faceVarying} \end{cases} $$

对一个 1000 三角形的纯三角网格：faceVarying UV 会有 3000 条目，而 vertex UV 只有顶点数（通常 500–800 个）条目，可用 indices 压缩。

indices 压缩

当多个面角共享同一个 UV 时，faceVarying primvar 允许用 indices 去重：数据数组只存唯一 UV，indices 数组长度为面角点总数，每项指向数据数组中的某一条目。

indexed_uv.usda
texCoord2f[] primvars:st = [
    (0,0), (1,0), (1,1), (0,1)    # 只有 4 个唯一 UV
]
int[] primvars:st:indices = [
    0, 1, 2,   # face 0 角点 → uv 0,1,2
    0, 2, 3    # face 1 角点 → uv 0,2,3
]
uniform token primvars:st:interpolation = "faceVarying"

在 ConvertAsset 的 glb/usd_mesh.py 里，读取时先检查 GetIndices()，若不为空就用索引展开数据，再判断插值模式决定是否需要"顶点分裂"（下一节专讲）。

小结 primvar = 数据 + 插值模式 + 可选 indices。faceVarying 是 UV 的默认模式，它允许每个面角点有独立 UV（解决 UV 缝合线问题），但 glTF 不支持——这是 USD→GLB 管线里的核心挑战，详见 7-4。
