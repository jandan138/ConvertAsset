UV (st) 与 normals

USD 网格上的 UV 坐标存在名为 primvars:st 的 primvar 里；normals 则是 UsdGeom.Mesh 的一个独立属性。两者都可以用不同的 interpolation 模式存储，但各自有惯例，也各自有坑。理解这些细节，是后续 GLB 导出不丢 UV / 法线的基础。

From GAMES101 在 GAMES101 的纹理映射（texture mapping）章节里，你把 UV 坐标绑定到顶点，再用重心坐标插值。这对应 vertex 插值。但现实资产中，UV 缝合线（seam）要求同一顶点在不同面上有不同 UV——这就需要 faceVarying。

UV：primvars:st

USD 的 UV 坐标没有硬编码的属性名；行业惯例是用 primvars:st（来自 RenderMan 的命名传统，\(s\) = 水平，\(t\) = 垂直）。ConvertAsset 的所有读写代码都以 "st" 作为 primvar 名称查询。

典型的 faceVarying UV 在 USDA 里长这样：

uv_example.usda
def Mesh "Box" {
    texCoord2f[] primvars:st = [
        (0,0), (1,0), (1,1), (0,1),   # 底面 UV
        (0,0), (1,0), (1,1), (0,1),   # 侧面 UV（可能完全相同，但角点独立）
        # ...更多面角点...
    ]
    uniform token primvars:st:interpolation = "faceVarying"
}

UV 坐标的 \(v\) 轴方向：USD（与 OpenGL）约定左下为 \((0,0)\)，而 glTF / DirectX 约定左上为 \((0,0)\)。导出 GLB 时需要对 \(v\) 做翻转：\(v' = 1 - v\)。ConvertAsset 的 glb/usd_mesh.py 目前直接传递，具体翻转由材质采样器的 wrap 模式处理。

normals：法线

normals 是 UsdGeom.Mesh 上的一个标准属性（不是 primvar），存储各顶点或面角的法线向量。常见插值模式：

  • vertex：每个顶点一条法线（平滑阴影），数量 = len(points)。
  • faceVarying：每个面角一条法线（硬边 / 折痕），数量 = ∑ faceVertexCounts。
  • uniform：每个面一条法线（faceted，平面阴影）。

ConvertAsset 的 glb/usd_mesh.py 当前只读取 vertex 法线（条目数与 points 相等时），其他模式的法线暂时忽略：

glb/usd_mesh.py（节选）
# 仅处理顶点法线：数量与顶点数一致的情况
normals = usd_mesh.GetNormalsAttr().Get()
if normals and len(normals) == len(points):
    normals_np = np.array(normals, dtype=np.float32)
else:
    normals_np = None   # faceVarying 或 uniform 法线暂不处理

faceVarying UV 的"顶点分裂"预览

当 st 的插值为 faceVarying 时，不同面上同一顶点可能有不同 UV（缝合线处）。glTF 要求每个顶点只能有一个 UV，所以必须"分裂"顶点——把共享位置但 UV 不同的面角点变成独立顶点。

橙色顶点在两个面上 UV 不同 → 必须分裂为两个独立顶点
