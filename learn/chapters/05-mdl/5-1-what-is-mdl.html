MDL 是什么：declarative + procedural

在 GAMES101 里，"材质"往往意味着一个手写的着色函数：你可以把任意计算塞进去。MDL（Material Definition Language）则换了一种思路——它把材质拆成两层：一层用声明式（declarative）语言描述"光与表面如何交互"的物理接口，另一层用过程式（procedural）语言编写任意复杂的参数计算函数。两者结合，让材质既对渲染器透明，又保留艺术家的无限灵活性。

两层设计：声明 vs 过程

NVIDIA 在 MDL 1.1 技术文档中将这两层描述得非常清晰：

  • 声明式材质定义——定义"计算什么"（what to compute），而非"如何计算"（how to compute）。材质的物理属性（BSDF、EDF、体积散射等）用固定的内建构件组合表达，渲染器可以分析这个结构并做路径追踪/重要性采样等优化。
  • 过程式函数语言——保留了类 C 的函数编写能力，用于计算材质参数的值（如贴图查找、程序化噪声、颜色混合等）。这些函数是纯函数（pure functions），无副作用、无全局依赖，便于 GPU 并行调度。
与传统着色语言的对比 GLSL / HLSL / OSL 等着色语言把所有三件事混在一起：纹理计算、材质参数描述、以及渲染算法本身。MDL 明确只做前两件，把第三件留给渲染器。这让渲染器可以"理解"材质而不是把它当作黑盒执行。

材质结构体：material struct

MDL 最核心的类型是内建的 material struct，它描述一个材质与光交互的所有物理属性：

material_struct.mdl
// MDL 内建结构体——声明一个材质的物理接口
struct material {
    uniform bool thin_walled = false;
    material_surface surface  = material_surface();
    material_surface backface = material_surface();
    uniform color    ior      = color(1.0);      // 折射率
    material_volume  volume   = material_volume();
    material_geometry geometry = material_geometry();
};

// surface 子结构：BSDF（散射）+ EDF（发光）
struct material_surface {
    bsdf             scattering = bsdf();
    material_emission emission  = material_emission();
};

这个结构体是 MDL 的渲染器接口——渲染器只需要读懂这个结构，就可以实现路径追踪、双向散射、体积渲染等各种算法，不需要执行任何用户代码。用户代码（过程式部分）仅用于为这些字段提供参数值。

模块化与复用：module

MDL 的另一个关键设计是 module（模块）系统。每一个 .mdl 文件就是一个模块，定义自己的命名空间，包含材质定义、函数、类型和常量。模块之间可以 import，使大型材质库（如 vMaterials）得以按主题分层组织。

下面是一个非常简单的 MDL 材质示例，展示声明式与过程式的配合：

simple_diffuse.mdl
mdl 1.5;
import ::df::*;      // distribution functions 模块
import ::state::*;   // 渲染器状态（法线、UV 等）
import ::tex::*;     // 纹理查找

// 过程式函数：从贴图计算 baseColor（可并行执行）
color get_base_color(uniform texture_2d tex) {
    return tex::lookup_color(tex, state::texture_coordinate(0).xy);
}

// 声明式材质：指定 surface.scattering 用 diffuse_reflection_bsdf
export material simple_diffuse(
    uniform texture_2d base_color_tex = texture_2d()
) = material(
    surface: material_surface(
        scattering: df::diffuse_reflection_bsdf(
            tint: get_base_color(base_color_tex)
        )
    )
);
小结 MDL = 声明式材质结构（让渲染器理解物理接口）+ 过程式纯函数（让艺术家自由编写参数计算）。这个设计比传统着色语言对渲染器更"透明"，但代价是：它对渲染器有依赖——这正是下一节要讨论的 renderer-bound 问题的根源。
