The main paper uses four evaluation checks: proxy preservation, GRScenes VLM
grounding, material-effect diagnostics, and embodied-stack usability. The
purpose of this appendix is to make the scope explicit. Each check answers a
different question. Keeping them separate makes clear which measurement surface
supports each result.

Table~\ref{tab:acl_evidence_gate_registry} gives the shortest reading guide.
Proxy render scores cover matched cameras and lighting. VLM rows cover the
specified prompt, coordinate format, and target set. Material panels make
selected failure modes visible. The InternNav run checks whether matched scenes
can be exercised by the official stack. Presenting the same result as a table,
derivation, and visual example is useful here because the paper uses several
measurement surfaces: metrics, prompts, material cases, and navigation media.

\subsection{Reading Order}

Appendix~\ref{sec:supp-render-atlas} starts with the render atlas
so readers can inspect the actual image evidence before the formal details.
Appendix~\ref{sec:supp-derivations} gives the mathematical definitions used by
the evaluation checks. Appendix~\ref{sec:supp-vlm-protocol} expands the prompt and
coordinate protocol. Appendix~\ref{sec:supp-grscenes-visuals} collects
additional GRScenes diagnostics. Appendix~\ref{sec:supp-material-effects}
summarizes material-effect and NVIDIA-baseline risk cases.
Appendix~\ref{sec:supp-internnav-visuals} is the largest navigation section; it
shows selected stills and explains the media inventory.
Appendix~\ref{sec:supp-theory} records mechanism-level hypotheses and
failure-mode interpretations. Appendix~\ref{sec:supp-reproducibility} lists the
artifact and media sources. The appendix supports the main-paper results;
broader claims about lossless MDL semantics, new VLM backends,
population-level material rates, or general navigation robustness would require
separate experiments.
