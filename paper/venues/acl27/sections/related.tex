\section{Related Work}
\label{sec:related_work}

This paper links VLM grounding, embodied simulation, and rendered 3D
evaluation. All three rely on rendered assets as inputs. Yet conversion is
usually treated as setup, not as part of the experiment.

\paragraph{Grounding and VLN.}
Vision-language navigation maps instructions to views and paths.
Room-to-Room introduced this task in indoor scenes~\cite{Anderson2018Vision},
and REVERIE adds remote object
search~\cite{Qi2020REVERIE}. Recent VLMs expose grounding outputs:
Shikra supports points~\cite{Chen2023Shikra}, and Ferret
supports grounding~\cite{You2024Ferret}. We add
no new VLM model or VLN benchmark. We study the measurement
question behind them: whether a converted synthetic scene retains the same
answer and point-grounding evidence as the source render.

\paragraph{Embodied simulation and synthetic scene benchmarks.}
Habitat, \mbox{AI2-THOR}, and ThreeDWorld provide interactive testbeds for
agents that use vision and language~\cite{Savva2019Habitat,Kolve2017AI2THOR,Gan2021ThreeDWorld}.
Isaac Sim and Isaac Lab~\cite{Mittal2023Orbit,Mittal2025Isaac} supply a
robotics stack built on USD scenes, RTX, and MDL graphs. \mbox{GRUtopia} and
GRScenes add city scenes for robots, and \mbox{InternNav/DualVLN} adds a
foundation-model route for VLN~\cite{Wang2024GRUtopia,Wei2026Ground}. We use
them as test substrates, not rankings. For KuJiaLe, matched scenes give entry
evidence for the tested setting rather than a broad navigation result.

\paragraph{Synthetic rendering shifts.}
{\hyphenpenalty=10000\exhyphenpenalty=10000\relax
Prior transfer work treats rendering as a task factor. Random texture and
light changes support transfer~\cite{Tobin2017Domain,Tremblay2018Training};
realistic rendering can also help vision models~\cite{Zakharov2022Photo}.
SynTable and Synthetica use Isaac/Omniverse pipelines for robot
data~\cite{Ng2023SynTable,Singh2025Synthetica}. Our case is narrower: MDL is
rewritten to USD PreviewSurface with scene content and task labels fixed where
the protocol allows. We ask whether the asset still matches after that
toolchain step.\par}

\paragraph{Proxy metrics versus task evidence.}
Perceptual and feature metrics such as SSIM~\cite{Wang2004Image},
LPIPS~\cite{Zhang2018Unreasonable}, CLIP~\cite{Radford2021Learning}, and
DINOv2~\cite{Oquab2023DINOv2} are proxy checks. They test whether pixels or
features stay close after the rewrite. The task asks a stricter test: do prompt
rules, the parser, point frame, and target cue expose the same evidence to a
model? Proxy match is only a first proxy check. Answer stability, point hits,
frame checks, material-risk bins, and stack checks stay separate.
