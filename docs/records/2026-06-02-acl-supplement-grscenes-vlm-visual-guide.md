# 2026-06-02 ACL Supplement GRScenes VLM Visual Guide

This record documents the supplement iteration that adds a render-heavy
GRScenes/VLM visual guide and reduces a table-float layout issue in the
GRScenes diagnostic appendix.

## Context

The active supplement goal is to make the appendix less sparse and more
render-evidence driven. The VLM/GRScenes pages were a clear weak point: pages
13--22 were mostly tables, equations, or short explanatory paragraphs, with
several pages below 5% active rendered-page density. A local audit found the
GRScenes diagnostic section especially vulnerable to a "text then table" float
ordering problem after adding a full-width figure.

## Design

I used the image-generation skill once for a non-evidence layout reference:

- `$CODEX_HOME/generated_images/019e5ded-e82b-7c03-91bd-2a636674e25e/ig_075eb20710f7ed46016a1e8a3dbbe481939d7ec36a39fb6ef7.png`

That generated image was not promoted into the paper. It only informed the
layout shape: grouped render rows, compact table-link cues, and a
normalized-1000 protocol row.

The tracked paper figure uses only registered project evidence:

- clean retake render frames from
  `paper/shared/evidence/raw/grscene_vlm_grounding/retake_zoom_renders/`
- registered target boxes from
  `paper/shared/evidence/raw/grscene_vlm_grounding/retake_zoom_expanded30_target_projection_qa_report.json`
- GRScenes context crops from `fig_grscene_qualitative.png`,
  `fig_render_scene_evidence_wide.png`, and `fig_supplement_render_atlas.png`

The old unsafe `fig_vlm_grounding_cases.png` red-material diagnostic panel is
not used.

the current tool rule allows subagents only when explicitly requested by the
user.

- Page 17: PASS. The new GRScenes VLM visual guide is a coherent render-heavy
